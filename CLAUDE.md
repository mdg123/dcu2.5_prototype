# 다채움 품질 제고사업 프로토타입

## 프로젝트 개요

충청북도교육청 통합 교육 플랫폼 "다채움"(dcu.cbe.go.kr)의 UI/UX 프로토타입 프로젝트이다.
개별적으로 제작된 HTML 프로토타입들을 실제 사이트의 네비게이션 구조처럼 연결하는 것이 목표이다.

- 대상 사용자: 교사, 학생, 학부모, 관리자
- 실제 사이트 분석 문서: `다채움 포털 분석보고서.md` (반드시 참고할 것)

## 기술 스택

### 공통 기술 특성
- **프레임워크 없음**: 순수 HTML/CSS/JavaScript (React, Vue 등 미사용)
- **CSS 방식**: 대부분 인라인 `<style>` 블록, 일부 외부 CSS 파일 사용
- **JS 방식**: 바닐라 JS, 외부 프레임워크 없음
- **반응형**: 모바일 브레이크포인트 768px, 1024px, 1200px 대응

### 외부 라이브러리 (CDN)
| 라이브러리 | 용도 | 사용 위치 |
|-----------|------|----------|
| Pretendard | 주 폰트 | 전체 |
| Noto Sans KR | 보조 폰트 | 일부 |
| Chart.js | 차트/그래프 | 클래스 홈, 학습분석, 오답노트 |
| Font Awesome 6.4.0 | 아이콘 | 오답노트 등 |
| Bootstrap Icons | 아이콘 | 포털 |
| Lucide Icons | 아이콘 | AI 맞춤 학습 |
| Quill Editor 1.3.6 | 리치텍스트 편집기 | 과제 |
| XLSX | Excel 파일 처리 | AI 맞춤 학습 |

### 공유 CSS 파일
- **dacheum-common.css**: 디자인 시스템 프레임워크 (CSS 변수, 유틸리티 클래스, 기본 컴포넌트)
  - 사용처: 클래스 관리, 클래스 찾기, 클래스별 학습분석
- **dacheum-style.css**: 나의 클래스 전용 스타일 (헤더, 탭, 카드, 필터, 모달, 캘린더)
- **styles.css (오답노트)**: 오답노트 전용 대시보드 스타일

## 디자인 시스템

### 색상 체계
```
Primary 블루:   #2563eb, #1d4ed8, #3b82f6
Success 그린:   #10b981, #4CAF50, #16a34a
Warning 오렌지: #f59e0b, #FF9800
Danger 레드:    #ef4444, #dc2626
Purple 보라:    #8b5cf6, #7c3aed
Pink 핑크:      #ec4899, #e91e63

배경:           #F5F7FA (페이지), #FFFFFF (카드)
텍스트 Primary: #1A1A2E, #1f2937
텍스트 Muted:   #6B7280, #64748b
```

### 타이포그래피
- 폰트: Pretendard (주), Noto Sans KR (보조), system-ui 폴백
- 제목: 16px~28px, weight 600~700
- 본문: 13px~14px, weight 400~500
- 캡션: 11px~12px, weight 500~600

### 공통 UI 컴포넌트 패턴
- **카드**: 흰 배경, border-radius 8~16px, 미세한 그림자, padding 16~24px
- **버튼**: Primary(블루) / Secondary(회색) / Outline(테두리) / Ghost(투명)
- **모달**: 반투명 오버레이(rgba 0,0,0,0.5)) + 중앙 카드 + 닫기(×) 버튼
- **테이블**: 헤더 회색 배경, 행 hover 효과, 상태 배지 컬러 코딩
- **탭**: 수평 탭 바, 활성 탭 하단 컬러 언더라인 또는 배경색
- **필터**: 상단 검색바 + 셀렉트 드롭다운 + 토글 칩
- **원형 게이지**: 이수율/제출률/풀이율 표시
- **차트**: Chart.js 기반 (bar, pie, line, doughnut)

### 레이아웃
- 헤더: 높이 60px, 좌측 로고 + 우측 아이콘
- 서브헤더(GNB): 높이 48px, 수평 메뉴
- 컨텐츠: 최대 1200~1440px, 좌우 패딩 24px
- 사이드바: 우측 240~280px (나의 클래스, 오늘의 학습 등)

## 프로토타입별 특성

### 파일 구조 유형

**타입 A: 단일 HTML (인라인 CSS/JS)** — 대부분의 프로토타입
- 하나의 HTML 파일에 `<style>`과 `<script>` 모두 포함
- 독립 실행 가능, 외부 의존성 CDN만 사용

**타입 B: HTML + 외부 CSS/JS 분리**
- 나의 클래스: dacheum.html + dacheum-style.css + dacheum-script.js
- 오답노트: index.html + styles.css + script.js

**타입 C: HTML + 공통 CSS 프레임워크**
- 클래스 관리, 클래스 찾기, 클래스별 학습분석: 각각 dacheum-common.css 사용

### 프로토타입 복잡도

| 수준 | 파일 | 특징 |
|------|------|------|
| 매우 높음 | AI 맞춤 학습 (learning_map) | 캔버스 렌더링, 드래그/줌, Excel 파싱 |
| 높음 | 포털 메인, 클래스 홈, 평가 기능, 오답노트, 학습분석 | 다중 섹션, 차트, 인터랙션 |
| 중간 | 나의 클래스, 클래스 관리, 클래스 찾기, 콘텐츠 v2, 오늘의 학습, CBT 전체서비스 | 필터/검색/목록/모달 |
| 보통 | 수업 플레이어, 과제, 알림장, 출석부, 성장기록, 포트폴리오, 평가지플레이어 | 단일 기능 집중 |

### 특이사항
- **알림장**: 5가지 테마 변형 지원 (칠판, 코르크, 화이트보드, 파스텔, 클래식)
- **과제**: 교사/학생 역할 전환 UI 구현
- **오답노트**: 교사 뷰/학생 뷰 분리 구현
- **출석부**: CSS 애니메이션 (popIn, pulse) 활용
- **AI 맞춤 학습**: 유일하게 Canvas API + 파일 업로드(XLSX) 사용
- **채움콘텐츠**: SPA 단일 파일 (해시 기반 라우팅, GNB 2차/3차 메뉴 연동)
- **채움CBT**: 통합본 2개 (전체 서비스용 / 평가지 플레이어용)

## 작업 규칙

### 필수 준수 사항
1. **한국어 UI**: 모든 텍스트, 레이블, 플레이스홀더는 한국어로 작성
2. **기존 스타일 유지**: 수정 시 해당 파일의 기존 색상/폰트/레이아웃 패턴을 따를 것
3. **파일명 주의**: 파일명에 공백과 괄호가 포함되어 있으므로 링크 시 URL 인코딩 필요
4. **CDN 의존성**: 새 라이브러리 추가 시 CDN으로 불러오기 (로컬 설치 불필요)
5. **더미 데이터**: 실제 교육 맥락에 맞는 데이터 사용 (과목명, 학생이름, 학교명 등)
6. **반응형 대응**: 최소 768px 브레이크포인트 지원

### 네비게이션 연결 작업 시
- 각 프로토타입에 공통 GNB(상단 메뉴)를 삽입하여 페이지 간 이동 가능하게 할 것
- 실제 사이트의 메뉴 구조는 `다채움 포털 분석보고서.md` 참고
- 현재 페이지에 해당하는 메뉴를 활성(active) 상태로 표시
- 프로토타입이 없는 메뉴는 비활성 처리 또는 "준비 중" 표시
- 기존 프로토타입의 콘텐츠 영역은 변경하지 말 것
- 상대 경로로 파일 간 링크 연결

### 수정 작업 시
- 원본 파일 수정 전 백업 생성 권장
- dacheum-common.css를 사용하는 파일은 해당 디자인 시스템 변수 활용
- Chart.js 사용 파일 수정 시 기존 차트 설정 패턴 유지
