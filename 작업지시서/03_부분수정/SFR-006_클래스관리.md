## 기존 구현 현황

### 현재 보유 기능
- ✓ 클래스 테이블: 클래스명, 상태 배지(활성/종료), 게시판유형 배지
- ✓ 3단계 클래스 복사 모달: 기본정보 → 게시판 선택 → 최종확인 (스텝 인디케이터)
- ✓ 게시판 추가 모달: 유형 선택(드롭다운), 토글(필수/선택)
- ✓ 3탭 설정 모달: 기본정보/게시판/멤버관리
- ✓ 게시판 리스트: 드래그 가능 정렬

## 추가/보완 항목

### 1. 복사 모달 - 항목별 선택 체크박스 세분화
**현재 상태**: 복사 마법사에서 "게시판 복사" 옵션만 존재
**개선 사항**:
- 2단계 (기본정보 입력) 후, 3단계 (항목 선택)로 세분화:
  - [ ] 기본 게시판 복사
  - [ ] 커스텀 게시판 복사
  - [ ] 게시판 설정 (필수/선택 구분) 복사
  - [ ] 클래스 소개글/배너 이미지 복사
- 각 체크박스 옆 설명 텍스트 추가 (11px, #6B7280)
  - 예: "기본 게시판 - 공지, 자료실, 과제, 평가"
  - 예: "커스텀 게시판 - 선생님이 추가한 게시판들"

### 2. 멤버 복사 옵션 추가
**현재**: 복사 모달에 멤버 관련 옵션 없음
**추가 필요**:
- 복사 대상 단계에서 별도 섹션: "멤버 설정"
  - [ ] 학생 명단 복사 (기존 참여학생 자동 추가)
  - [ ] 학생 권한 설정 복사 (실명/익명 등)
  - [ ] 운영자/조교 복사
- 선택 안 한 경우 기본값: "새 클래스는 빈 상태로 생성"

### 3. 멤버 관리 탭 UI 강화
**현재 상태**: 기본 멤버 목록 표시만 가능
**개선 사항**:

#### 3-1. 승인/거절 상태 분리
- 탭 추가: "가입 신청 (5)", "승인됨 (45)", "거절됨 (2)"
- 각 탭 별 멤버 목록 필터링
- 신청 상태 배지 색상 코드:
  - 대기: #f59e0b (오렌지)
  - 승인: #10b981 (그린)
  - 거절: #ef4444 (레드)

#### 3-2. 일괄 작업 버튼 추가
- "가입 신청" 탭에서 선택된 멤버 체크박스
- 버튼: "선택 승인 (5)", "선택 거절 (5)", "모두 승인"
- 액션 시 확인 모달: "5명의 신청을 승인하시겠습니까?"

#### 3-3. 멤버별 권한 설정
- 각 멤버 행의 우측에 "권한 변경" 드롭다운:
  - 학생 (기본)
  - 조교
  - 운영자
- 선택 시 확인: "역할을 변경하시겠습니까?"

### 4. 멤버 검색/필터 추가
- 멤버 목록 상단에 검색창 추가 (placeholder: "이름 또는 학번 검색")
- 필터 칩: "활동중", "비활동", "최근가입순"
- 멤버 테이블 좌측에 체크박스 (일괄 선택 헤더 포함)

## 구현 체크리스트

### 단계 1: 복사 모달 항목 체크박스 세분화
- [ ] 3단계 "항목 선택" 새 스텝 추가
- [ ] 체크박스 4개 (기본게시판, 커스텀게시판, 게시판설정, 소개글/배너) 구현
- [ ] 각 체크박스 옆 설명 텍스트 추가
- [ ] 최종 확인 단계에서 선택된 항목 요약 표시

### 단계 2: 멤버 복사 옵션 추가
- [ ] 복사 모달 "멤버 설정" 섹션 추가 (3개 체크박스)
- [ ] 기본값 설정: "새 클래스는 빈 상태로 생성"
- [ ] 최종 확인 단계에 멤버 복사 내용 표시

### 단계 3: 멤버 관리 탭 구조 변경
- [ ] 3개 탭 (가입신청/승인됨/거절됨) 구현
- [ ] 각 탭 별 필터링 로직 추가
- [ ] 배지 색상 코드 적용 (대기/승인/거절)

### 단계 4: 멤버 관리 기능 구현
- [ ] 체크박스 선택 UI 추가 (헤더 일괄 선택 포함)
- [ ] 선택 멤버 일괄 승인/거절 버튼 추가
- [ ] 확인 모달: "N명 승인하시겠습니까?" 메시지
- [ ] 멤버별 권한 변경 드롭다운 추가 (학생/조교/운영자)
- [ ] 권한 변경 확인 모달 구현

### 단계 5: 검색/필터 기능
- [ ] 멤버 목록 상단 검색창 추가
- [ ] 필터 칩 (활동중/비활동/최근가입순) 추가
- [ ] 검색 및 필터 조합 로직 구현

### 단계 6: 테스트 및 검증
- [ ] 모달 스텝 인디케이터 진행 상황 확인
- [ ] 일괄 작업 후 멤버 목록 업데이트 확인
- [ ] 토스트 메시지 (승인/거절/권한변경 완료) 표시 확인
- [ ] 모바일 반응형 테스트 (768px 이상)

## 주의사항

1. **기존 게시판 설정 탭 유지**: 설정 모달 3탭 구조 변경 금지
2. **드래그 정렬 유지**: 게시판 리스트의 드래그 기능 손상 금지
3. **모달 깊이 관리**: 확인 모달이 복사 마법사 위에 오버레이될 때 z-index 조정
4. **더미 데이터**: 멤버 목록은 20-50명 규모의 샘플 데이터 사용
5. **권한 관리 로직**: 현재 사용자가 "운영자"일 때만 권한 변경 가능하도록 제약
