## 기존 구현 현황

### 전체서비스 현황
- **5개 페이지**:
  1. 평가지 목록 (필터+검색)
  2. 나의 평가지 (학생이 응시해야 할 평가지)
  3. 나의 결과 (학생이 응시 완료한 결과)
  4. 교사 결과 (교사가 등록한 평가지 응시 결과)
  5. 평가지 등록 (4단계 등록마법사)
- **공통기능**: 필터, 검색, 모달/토스트 알림
- **응시 기능**: 타이머, 탭이탈감지, 자동저장,

### 평가지플레이어 현황
- **사이드바**: 문항목록(번호/상태), 진행률 게이지, 상태범례
- **메인 콘텐츠**: 문항카드(문제/선택지/답안입력)
- **문항유형 5가지**: 객관식단일선택, 객관식복수선택, 단답형, 복수단답형, 서술형
- **네비게이션**: 이전/다음 버튼, 문항 점프(사이드바 클릭)

### 디자인 톤
- 색상: 파란색 주(#2563eb, #3b82f6), 회색 중립
- 배경: #f5f7fa (페이지), #ffffff (카드)
- 폰트: Pretendard, 제목 16px/600, 본문 14px/400

## 추가/보완 항목

### 1. 메뉴 구조 재정비 (필수)
**기존**: 5개 페이지가 분산
**변경구조**:
```
채움CBT
├─ 출제관리 (교사 전용)
│  ├─ 평가지 목록
│  ├─ 평가지 등록
│  └─ 평가 결과 현황
├─ 응시관리 (학생 전용)
│  ├─ 나의 평가지 (대기 중)
│  ├─ 응시하기 (진행 중)
│  └─ 나의 결과 (완료됨)
└─ 결과확인 (공통)
   ├─ 응시 결과 조회
   └─ 성적표 다운로드
```
- GNB에서 3개 대메뉴 노출 (또는 학생/교사 역할에 따라 필터링)
- 각 메뉴 우측에 서브탭 표시

### 2. 시험모드/대기모드 전환 (필수)
**추가 필요**:
- **플레이어 상단**에 "모드 표시" 배지 추가:
  - "시험 응시 중 ⏱️" (타이머 활성, 제출 버튼 노출)
  - "임시저장 대기 중" (타이머 미표시, 자동저장 중)
- 모드 전환 토글 (교사 또는 관리자용): "시험모드" / "대기모드"
- 시험모드 진입 시 경고 팝업: "이 평가지는 시험 상황입니다. 탭이탈 감지가 활성화됩니다."

### 3. 채움콘텐츠 연계 표시 (권장)
**추가 필요**:
- 평가지 목록 또는 상세에서 **"관련콘텐츠" 섹션** 추가:
  - 이 평가지와 매핑된 채움콘텐츠 링크 표시
  - 아이콘(🎥) + "동영상 강의 3개" 링크
  - 예: "단원 3-2 함수 기본개념 [3개 콘텐츠 보기]"
- 링크 클릭 시 채움콘텐츠 페이지로 이동 (또는 새탭)

### 4. 결과 상세화 (필수)
**추가 필요**:
- **나의 결과** 또는 **결과확인** 페이지에서:
  - 기본 점수/등급 표시 (기존)
  - **상세 분석 추가**:
    - 성취기준별 정답률 (테이블: 성취기준 / 정답률 / 점수)
    - 영역별 정답률 (막대그래프: Chart.js)
    - 난이도별 정답률 (파이 또는 도넛 차트)
  - 문항별 풀이 기록 표시 (문항번호 / 정답 / 응답 / 소요시간)

### 5. 문항내용 인라인뷰어 (권장)
**추가 필요**:
- 플레이어의 **문항카드 상단**에 원본 콘텐츠 임베딩:
  - 이미지 자동스케일 (최대 가로 100%, 세로 400px)
  - 동영상: iframe 임베딩 (16:9 비율, 반응형)
  - 텍스트/문서: 인라인 표시 (스크롤 가능)
  - "원본보기" 버튼: 새탭에서 전체 콘텐츠 오픈

### 6. 결과 다운로드 XLSX (선택)
**추가 필요**:
- **결과확인 페이지 우측상단**에 "다운로드" 버튼 추가:
  - 버튼 클릭 시 선택지 표시:
    - "성적표 (엑셀)" → 학생별 점수/등급 목록
    - "상세결과 (엑셀)" → 문항별 정답/응답/소요시간 상세
    - "취약영역 (엑셀)" → 성취기준별/영역별 분석 데이터
  - XLSX 파일 생성 (Sheet.js 또는 유사 라이브러리 활용)

### 7. 답안 임시저장 (권장)
**추가 필요**:
- 플레이어에서 **"임시저장" 버튼** 추가 (상단 우측):
  - 현재 문항까지의 답안을 로컬저장 또는 서버저장
  - 저장 완료 시 "저장됨 ✓" 토스트 메시지 표시
  - 자동저장과 별도로 수동 저장 기능 제공
  - 응시 재개 시 이전 진행상태 복구

### 8. 네트워크 장애 복구 안내 (선택)
**추가 필요**:
- 응시 중 네트워크 끊김 감지:
  - 상단에 "⚠️ 연결 끊김. 자동복구 중..." 배너 표시
  - 일정 시간(10초) 후 자동 재연결 시도
  - 재연결 실패 시 "수동으로 연결 복구" 버튼 제공
  - 자동저장된 답안은 유지되도록 안내

## 디자인 참고

### 색상 정의
```
모드 배지:
- 시험중: #ef4444 (빨강) + 시계 아이콘
- 대기중: #f59e0b (주황) + 준비 아이콘

상태별 점수색:
- 100점: #10b981 (초록)
- 80점 이상: #3b82f6 (파랑)
- 60점 이상: #f59e0b (주황)
- 60점 미만: #ef4444 (빨강)
```

### 컴포넌트 명세
- **모드 배지**: 상단 중앙, 높이 32px, padding 8px 16px, border-radius 4px, 배경색 + 텍스트
- **관련콘텐츠 섹션**: 카드 형식, 좌측 아이콘(40x40px) + 텍스트 + 링크 화살표
- **상세분석 테이블**: 헤더 회색(#f3f4f6), 행높이 40px, 우측정렬
- **임시저장 버튼**: 높이 36px, 폭 120px, 테두리 스타일, 우측상단 고정

### 레이아웃 조정
- 플레이어 상단: 모드배지 + 타이머 + 임시저장 버튼 수평배치
- 결과 페이지: 기본정보(좌측) + 상세분석 차트(우측) 2단 레이아웃
