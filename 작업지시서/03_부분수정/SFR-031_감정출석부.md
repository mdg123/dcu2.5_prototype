## 기존 구현 현황
- 감정 선택 기능 존재 (체크인 시 감정 선택 화면)
  - 기본 감정: 😊/😐/😭 (3가지)
- 학생 뷰: 캐릭터, 경험치, 피드, 달력, 체크인 버튼
- 학생별 감정 이력은 UI에 미구현

## 추가/보완 항목

### 1. 감정 이모티콘 다양화
- 현재: 기본 3가지 감정만 제공
- **필요**:
  - 감정 옵션 확장: 5~7가지
    - 매우 좋음 😄 (초록색)
    - 좋음 😊 (파란색)
    - 보통 😐 (노란색)
    - 피곤 😴 (회색)
    - 안 좋음 😞 (주황색)
    - 매우 안 좋음 😢 (빨강색)
    - (선택) 화남 😠 (빨강색)
  - 각 감정별 컬러 코딩 (위 참고)
  - 이모티콘 크기: 48px

### 2. 감정 선택 이유 작성 입력란
- **필요**:
  - 감정 선택 후 "어떤 이유인가요?" 텍스트 입력창 추가
  - 플레이스홀더: "예: 어려운 과제를 끝냈어요", "최근 스트레스가 많아요" 등
  - 최대 100자 제한
  - 선택사항 (입력 없이 제출 가능)
  - 제출 시 감정 + 이유 함께 저장

### 3. 감정 회고 UI (주간/월간 타임라인)
- **필요**:
  - 학생 뷰: "감정 회고" 탭 또는 섹션 추가
  - 주간 타임라인:
    - 월~금 날짜 가로 표시
    - 각 날짜 아래 해당 감정 이모티콘
    - 이모티콘 클릭 → 그날의 감정 + 이유 팝업 표시
  - 월간 타임라인: 주간 확장 버전 (전체 월 표시)
  - 감정 추이 텍스트: "지난주 감정이 나아지고 있어요! 🌟"

### 4. 교사용 감정 현황 리포트 (감정 비율 변화 추이 차트)
- **필요**:
  - 교사 뷰에 새 섹션: "반 전체 감정 현황"
  - 파이/도넛 차트:
    - 현재 반 학생들의 감정 분포 (오늘)
    - 각 감정별 인원수 + 퍼센트
    - 범례: 색상코딩된 감정 라벨
  - 라인 차트: 주간/월간 감정 추이
    - X축: 날짜
    - Y축: 각 감정별 인원
    - 멀티라인 (좋음/보통/안좋음 등 3개 라인)
    - 트렌드 해석: "지난주 대비 긍정적 감정이 증가했습니다"
  - 기간 선택: "주간" / "월간" / "기간 선택"

### 5. 학생별 감정 이력 테이블
- **필요**:
  - 교사 뷰: "학생 감정 이력" 테이블 추가
  - 컬럼:
    - 학번 / 이름 / 최근 감정 / 이전 감정 / 추세 / 코멘트
  - 행 높이: 40px, hover 효과
  - 최근 감정 클릭 → 해당 학생의 감정 이력 상세 팝업
  - 정렬: 최근 감정순, 학번순, 이름순
  - 필터: "부정적 감정 중심 보기" (피곤/안좋음/매우안좋음)
