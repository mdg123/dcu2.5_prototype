### 2.3 영어과 (KICE) — 변칙 스키마 C (가장 복잡)

- **파일**: `(KICE) 교육과정 표준체계_영어과_2412_최종.xlsx`

**중학교(14열)**: 3단계/선행/연관/평가기준B~E 컬럼이 **빠진 형태** (ID~2단계, 성취기준코드, 성취기준, 평가기준 A·B·C·D·E 5단계). 성취수준 컬럼이 없고 `평가기준 A~E`로 **대체**된다는 점이 가장 큰 차이.

**고등학교(41열)** — 매우 넓음. 실제 데이터가 있는 컬럼은 A~V(22열) 정도고 W~AO(19열)는 공란. 평가기준이 **2블록 반복** 출현:

```
... | 평가기준(A) | (B) | (C) | (D) | (E) | 평가기준(A) | (B) | (C) | (D) | (E) | [공란 19개] ...
```

대표 3행(고영1 첫 3행)에서 앞 5개 평가기준 셀은 **공란**이고, 뒤 5개 평가기준에만 값이 들어 있음. → **추정**: 두 번째 블록이 "최종 평가기준"이고 앞 블록은 미사용(빈 틀로 남음)이거나, 이해영역/표현영역용 이중 기입 의도였으나 미채움.

**특이사항**
- 초·중·고 컬럼 수가 모두 다름(18/19 vs 14 vs 41) → **학교급별 파서 분기 필수**
- `3단계 내용 요소`, `선행 내용 요소`, `연관 내용 요소`가 **스키마에는 있으나 값은 거의 없음** (추정: 향후 채울 자리를 비워 둔 템플릿)
- 성취기준 1개 아래 2단계 요소가 수백 건 분기(초3-4 797행) → 데이터가 매우 조밀
- `평가기준` ≠ `성취수준` : 초등은 둘 다 존재(평가기준 공란), 중·고는 `평가기준` 컬럼만 있고 `성취수준` 헤더는 없음

- `선수학습ID → 후속학습ID` 로 **DAG 엣지가 명시적으로 들어가 있음** → 학습맵 그래프 구축의 1차 소스.
- ID 포맷이 2단계(`...CNN`) + 3단계(`...CNNDnn`) 의 2계층 접미사 구조. 다른 교과 파일엔 3단계ID가 없고 여기서만 D접미사로 확장됨.
- `요약` 시트는 R1부터 제목행/공란/설명행 혼재 — **병합셀 + 공란 패턴**. 데이터 시트(`학습맵_리니어연결`)는 깔끔함.
- `변경로그`/`변경로그_v2`/`미매칭_수동검토`는 **KICE 원본 vs KOFAC 기준 재매핑** 이력 — 시드 주입 시에는 무시 가능(참고 자료). `변경로그_v2`는 유사도 기반 자동매핑 결과를 담고 있음(유사도 0.83~0.955).

## 3. 공통 관찰

### 3.1 ID 포맷 일관성

모든 교과에서 2단계ID(최소 단위)는 다음 규칙을 따름:

```
[학교급문자][학년][과목코드3][A01][Bnn][Cnn]
 E = 초(Elementary)       KOR = 국어        A01 = 영역 순번(01~)
 M = 중(Middle)           SOC = 사회        Bnn = 1단계 순번
 H = 고(High)             ENG = 영어        Cnn = 2단계 순번 (리프)
                          PRA = 실과
                          SCI = 과학
                          INF = 정보
                          MAT = 수학
 학년 숫자:
   E4 = 초3-4학년군
   E6 = 초5-6학년군
   M3 = 중학교 (3년 묶음)
   H1/H2/H3 = 고 1/2/3학년
   E2 = 초1-2학년군 (수학 파일에서만 등장)
```

수학 학습맵 파일만 `Dnn` 접미사로 3단계ID를 확장.

### 3.2 성취기준 코드 포맷

- 초등: `[4국01-01]`, `[6사01-02]`, `[6실01-01]`
- 중학교: `[9영01-01]`, `[9과01-01]`, `[9기가01-01]`, `[9사(지리)01-01]`, `[9정01-01]`, `[2수01-01]`(수학파일의 초1-2)
- 고등학교: `[10공국1-01-01]`, `[10통사1-01-01]`, `[10공영1-01-01]`, `[10통과1-01-01]`, `[10통사2-01-01]`, `[12기가01-01]`, `[12정01-01]`

정규식(안): `\[(\d+)(공국|공영|통사|통과|기가|정|국|영|사|과|실|수)(?:\([^)]+\))?\d+(?:-\d+)?-\d+\]` — **중학교 사회(지리) 괄호** 와 **고등학교 2단 번호(`-01-01`)** 양쪽을 허용해야 함.

### 3.3 계층 깊이

| 파일 | depth 0 (영역) | depth 1 (1단계) | depth 2 (2단계) | depth 3 (3단계) |
|---|---|---|---|---|
| 국어·사회·실과·과학 | O | O | O (리프) | — |
| 영어 | O | O | O | 헤더만 (값 공란) |
| 정보 | O | O | O | **O (값 있음)** |
| 수학 학습맵 | O | O | O | **O (ID까지)** |

### 3.4 성취수준/평가기준 단계

| 파일·학교급 | 컬럼 | 단계 |
|---|---|---|
| 초등 전 교과(수학 제외) | 성취수준 A/B/C | 3 |
| 중·고 국어·사회·과학·실과 | 성취수준 A/B/C/D/E | 5 |
| 영어 초등 | 평가기준 A/B/C + 성취수준 A/B/C (평가기준 빈 값) | 3+3 |
| 영어 중학 | 평가기준 A/B/C/D/E (성취수준 컬럼 없음) | 5 |
| 영어 고등 | 평가기준 A~E 두 블록 (앞 블록 공란) | 5(×2) |
| 정보 | 성취수준 A~E | 5 |
| 수학 학습맵 | — | 없음 |

### 3.5 빈 행·병합셀 처리 주의점

- 모든 데이터 시트는 **1행 단일 헤더**라 병합셀 문제는 거의 없음.
- 예외: 수학 파일 `요약` 시트 (제목행 + 공란 + 설명 테이블) — 파싱 대상에서 제외.
- 영어 고등학교 시트는 R1 헤더가 19개만 채워지고 20번 이후 열 헤더가 공란 → `ws.max_column=41` 로 반환되지만 실제 유효 헤더는 22열까지.
- 실과 고등학교 시트는 성취수준 5컬럼이 모두 `""`(공란) — **NULL 허용 필수**.
- 사회·고2 일부 행은 성취수준 5컬럼이 모두 **같은 문자열**(원본 복붙 흔적) → 정규화 저장 시 하나만 저장하고 단계는 NULL로 둘지, 그대로 보존할지 정책 필요.

### 3.6 헤더 명칭 정규화 필요

같은 의미인데 띄어쓰기/괄호가 다르다:

| 표준(안) | 바리에이션 |
|---|---|
| `achievementStandardCode` | `성취기준 코드`(KICE), `성취기준코드`(정보·수학·KOFAC) |
| `contentArea` | `내용 체계 영역`(국어·사회·실과·과학·영어), `내용체계 영역`(정보), `내용체계영역`(수학) |
| `level1Element` | `1단계 내용 요소`, `1단계 내용요소`, `1단계내용요소` |

공백·가운뎃점(`·`) 유지, 괄호 표기(`평가기준(A)` vs `평가기준 (A)`) 혼재도 존재.

## 4. 파서 어댑터 권고

조사 결과에 기반한 **3개 어댑터 구성**을 권고:

1. **Adapter-Standard (A형)** : 국어·사회·실과·과학 (+ 정보의 A~G, I~O 부분)
   - 12열(초) / 14열(중·고) 고정 매핑.
   - 정보과는 3단계 컬럼 1개 추가만 예외 처리.
2. **Adapter-English (B형)** : 영어과 전용
   - 3단계·선행·연관·평가기준·성취수준 혼재. 고등학교는 2블록 평가기준 중 **데이터가 있는 쪽만** 채택하는 로직 필요.
   - 초·중·고 컬럼 수가 다르므로 **학교급별 서브 매퍼**.
3. **Adapter-MathLearningMap (C형)** : 수학 학습맵 전용
   - 성취수준 파싱 **하지 않음**.
   - `선수학습ID/후속학습ID` 를 `learning_map_edges(from_id, to_id)` 테이블로 적재.
   - `단원명/적용학년/적용학기` 를 `math_unit_info` 보조 테이블로 적재.
   - 3단계ID를 노드로, 2단계ID는 상위 그룹으로 보존.

정보과를 A형으로 묶을지(3단계 컬럼 추가 분기만) 별도 B형으로 쪼갤지는 코드 복잡도 취향 문제. 컬럼 수(12/14 기본 + 1/2)가 일관적이라 **A형 어댑터 안에서 조건 분기가 합리적**.
