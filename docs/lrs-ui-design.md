## 1. 디자인 시스템 감사 (Design Audit)

### 1.1 기존 다채움 디자인 토큰 추출

`public/css/common-nav.css`, `public/growth/class-dashboard.html`, `public/lrs/index.html`을 교차 분석한 결과 다채움은 이미 일관된 토큰 체계를 갖고 있다. 이를 LRS 전용으로 확장하되 변수명과 값은 최대한 보존한다.

#### (1) 색상 팔레트 — 유지

| 토큰 | 값 | 용도 |
|------|-----|------|
| `--primary` | `#2563eb` | 브랜드 메인, 활성 상태, CTA |
| `--primary-light` | `#dbeafe` / `#eff6ff` | 활성 배경, 뱃지 |
| `--primary-dark` | `#1d4ed8` | hover, pressed |
| `--secondary` | `#10b981` | 성공, 완료, 증가 |
| `--warning` | `#f59e0b` | 주의, 경고 |
| `--danger` | `#ef4444` / `#dc2626` | 오류, 부진, 감소 |
| `--purple` | `#8b5cf6` / `#9333ea` | 보조 액센트 |
| `--pink` | `#ec4899` | 보조 액센트 |
| `--gray-50 ~ 900` | `#f9fafb ~ #111827` | 10단계 중립색 |

#### (2) 타이포그래피

- 기본 패밀리: `'Pretendard','Noto Sans KR',system-ui,sans-serif`
- 사이즈 스케일(LRS 대시보드 기준):
  - Display: 28px/700 (KPI 숫자)
  - H1: 22px/700 (페이지 제목)
  - H2: 16px/700 (섹션 제목)
  - H3: 13px/700 (카드/차트 제목)
  - Body: 14px/500 · Small: 12px/500 · Caption: 11px/500
  - 숫자: `font-variant-numeric: tabular-nums`
- 자간: 제목류 `letter-spacing: -0.3px` ~ `-1.5px`(로고)

#### (3) 간격(Spacing) — 4px 그리드

`4 · 8 · 12 · 16 · 20 · 24 · 32 · 48` 8단계. 기존 코드의 padding/gap 분포와 일치.

#### (4) 라운딩

| 토큰 | 값 | 용도 |
|------|-----|------|
| `--radius-sm` | 6px | 버튼, 입력 |
| `--radius-md` | 10px | 카드, 섹션 |
| `--radius-lg` | 16px | 히어로 카드, 모달 |
| `--radius-pill` | 9999px | 뱃지 |

#### (5) 그림자

```
--shadow-sm: 0 1px 3px rgba(0,0,0,.08);
--shadow-md: 0 4px 6px -1px rgba(0,0,0,.10);
--shadow-lg: 0 10px 15px -3px rgba(0,0,0,.10);
--shadow-focus: 0 0 0 3px rgba(37,99,235,.25); /* 키보드 포커스 */
```

### 1.2 LRS 전용 확장 — 데이터 시각화 팔레트 10색

기존 브랜드 블루와 충돌하지 않으면서, 색맹(적록/청황) 대응이 가능한 Okabe–Ito 기반 10색 팔레트를 `--chart-1 ~ 10`으로 추가한다. 스택/다중 시리즈에서 인접 색상이 명도로도 구분되도록 정렬했다.

| 토큰 | HEX | 권장 용도 |
|------|-----|----------|
| `--chart-1` | `#2563eb` | 주 시리즈(브랜드) |
| `--chart-2` | `#10b981` | 양호/긍정 |
| `--chart-3` | `#f59e0b` | 중간/주의 |
| `--chart-4` | `#ef4444` | 부정/부진 |
| `--chart-5` | `#8b5cf6` | 보조 시리즈 A |
| `--chart-6` | `#0ea5e9` | 보조 시리즈 B(연한 청록) |
| `--chart-7` | `#f97316` | 보조 시리즈 C(오렌지) |
| `--chart-8` | `#14b8a6` | 보조 시리즈 D(틸) |
| `--chart-9` | `#a855f7` | 보조 시리즈 E |

성취수준용 발산 스케일: `우수(#10b981) · 보통(#f59e0b) · 미흡(#ef4444)`에 각 light 버전을 결합해 6단계 표현.

### 1.3 다크모드 판단

**결론: 초기 버전에는 도입하지 않는다.** 초등 교실 현장은 주간 밝은 조명이 기본이며, 교사용 모니터와 학생용 태블릿 모두 라이트 테마를 유지한다. 대신 **"차트 가독성 모드"**(배경 `#f8fafc`, 격자선 명도 강화)만 옵션으로 제공한다. 다크모드 도입은 관리자 뷰 2차 릴리즈에서 재검토.

## 2. IA (Information Architecture) 재설계

### 2.1 현재 IA의 문제

현재 7개 서브메뉴(홈/학습현황/학업성취/학습수행/맞춤학습/교사활용지수/일일현황)는 **데이터 관점**에서 묶여 있고 **사용자 관점**에서 묶여 있지 않다. 학생은 "맞춤학습"을 봐도 자기 데이터가 없고, 교사는 "교사활용지수"가 자기 수업 운영에 바로 연결되지 않는다.

### 2.2 역할(Role) 기반 상위 IA

상단에 **롤 스위처**를 배치하고(관리자만 3개 모두, 교사는 학생·교사, 학생은 학생만 가시), 좌측 서브메뉴는 롤에 따라 동적으로 바뀐다.

```
[학생 뷰]                    [교사 뷰]                       [관리자 뷰]
├ 내 학습 요약 (랜딩)        ├ 우리반 현황 (랜딩)            ├ 전체 통계 (랜딩)
├ 내 성취                    ├ 학생별 드릴다운               ├ 기간 비교
├ 내 추이                    ├ 경고 (부진/결손)              ├ 데이터 품질
└ 비교 (익명)                ├ 수업 활용도                   └ 내보내기 / 관리
                             └ 교과별 진단
```

### 2.3 각 뷰의 "랜딩 → 드릴다운 → 액션" 플로우

#### (A) 학생 뷰

```
[내 학습 요약]
  KPI 4장(이번주 학습시간/완료과제/평균 성취/연속일)
  → 최근 7일 스파크라인
  → "내 성취" 클릭
      ↓
  [내 성취] 교과별 성취수준 누적바 + 약점 성취기준 Top 5
  → 약점 카드 클릭 → [스스로채움 추천 콘텐츠로 이동] (액션)
  → "추이 보기" → [내 추이] 월별 라인
  → "내 반과 비교" → [비교] 익명 분포 바이올린/박스플롯
```

액션은 반드시 **다채움 내 다른 서비스로의 딥링크**로 종결한다. 대시보드가 종점이 되면 학생의 행동 변화를 못 이끈다.

#### (B) 교사 뷰

```
[우리반 현황]
  KPI 4장(학급 평균 성취/이번주 활동량/경고 학생수/미제출 과제)
  + 히트맵(요일×교시 활동 밀도) + 학생 리스트 (위험도 정렬)
  → 학생 행 클릭 → [드릴다운 사이드 드로어] (학생 상세 요약, 5초 이내)
  → "자세히" → [학생별 드릴다운] 전체 이력, 성취기준 맵
  → [경고] 탭: 3일 이상 미학습/연속 오답/결손 성취기준 Top N
      → 경고 카드의 [메시지 보내기] · [과제 부과] · [맞춤학습 추천] 버튼 (액션)
  → [수업 활용도]: 차시별 참여율, 콘텐츠 TOP, 평가 응답률
```

#### (C) 관리자 뷰

```
[전체 통계]
  기관/학년/학급 단위 스택바, 전체 MAU, 서비스별 활용시간
  → [기간 비교] 쌍대기간(YoY, 저번학기) 비교 차트
  → [데이터 품질] 로그 스키마 결측률, Ingest 실패율, 최근 집계 시각
  → [내보내기] Excel/CSV, 필터 기반 즉시 내보내기 + 예약
```

### 2.4 전역 네비게이션 규칙

- 좌측 서브메뉴(200px)는 **현재 롤의 메뉴만** 표시
- 상단 브레드크럼: `LRS > [롤] > [현재 페이지]`
- 상단 우측 "최근 동기화: HH:MM" 타임스탬프 항상 표기(데이터 신뢰성 신호)

- 학생 행 호버: 배경 `gray-50`. 행 우측 `→` 버튼 클릭 시 오른쪽에서 드릴다운 드로어(width 420px) 슬라이드 인
- 인사이트 카드의 [맞춤학습 배정] 클릭 → 확인 모달 → 채움클래스 과제 생성 API 호출

## 4. 재사용 컴포넌트 사양 (10+)

모든 컴포넌트는 HTML + CSS 클래스 + 바닐라 JS(≤200줄)로 구현 가능하도록 정의한다. 명명은 `dc-` 접두사(다채움).

### 4.1 `<dc-kpi-card>` — KPI 카드

**Props** (data-* 속성)
- `data-label`(string), `data-value`(number|string), `data-unit`(string)
- `data-trend`(up|down|flat), `data-delta`(string, 예: "+3%"), `data-compare-label`("지난주 대비")
- `data-spark`(JSON array, optional) — 20개 이하의 숫자 배열로 스파크라인 렌더
- `data-loading`(bool), `data-tone`(primary|success|warning|danger)

**상태**: default, hover(shadow-md→lg), loading(skeleton), empty("데이터 없음")

**구조**
```html
<article class="dc-kpi-card" data-tone="primary">
  <header class="kpi-label">평균 성취</header>
  <div class="kpi-value">82<span class="kpi-unit">점</span></div>
  <footer class="kpi-trend up">▲ 3점 · 지난주 대비</footer>
  <canvas class="kpi-spark" aria-hidden="true"></canvas>
</article>
```

**상호작용**: 전체 카드가 링크(role="link", tabindex=0, Enter/Space로 활성). 포커스 시 `shadow-focus` 링.

### 4.2 `<dc-chart-wrapper>` — 차트 래퍼

**Props**: `title`, `subtitle`, `period`, `legend-inline`(bool), `exportable`(bool), `insight`(string|null)

**구조**
```
┌ Title ──────────── [기간▾] [내보내기↓] [⋯] ┐
│ Subtitle                                      │
├──────────────────────────────────────────────┤
│          <canvas id="chart-xxx">             │
├──────────────────────────────────────────────┤
│ Legend (inline chip형)                       │
│ 💡 Insight: "수요일 활동이 평균 대비 30% 낮음"│
└──────────────────────────────────────────────┘
```

**상태**: loading(shimmer), empty(일러스트 + "선택한 조건에 데이터가 없어요"), error(원인 + 재시도)

### 4.3 `<dc-data-table>` — 데이터 테이블

**Props**: `columns`(name, key, sortable, type), `rows`, `page-size`(10/20/50), `selectable`(checkbox), `sticky-first`(bool), `row-click`(cb)

**기능**
- 열 헤더 클릭: 3-state 정렬(asc/desc/none), `aria-sort` 갱신
- 검색(상단 인풋, debounce 200ms), 페이지네이션(숫자+이전/다음)
- 반응형: 768px 이하 첫 열 고정 + 수평 스크롤, 360px는 카드 리스트로 변환 (`data-mobile="card"`)
- Export: CSV/XLSX 버튼은 `dc-chart-wrapper`와 동일 위치

### 4.4 `<dc-filter-bar>` — 필터 바

**Props**: 필드 배열 (type: date-range, select, multi-select, chip-toggle)

**상호작용**
- "적용" 버튼 클릭 시만 쿼리 갱신(성능). 단, 단일 토글은 즉시 적용
- "저장된 뷰" 드롭다운: 현재 필터 조합을 localStorage에 저장/불러오기(최대 5개, 이름 입력)
- 모바일(768px 이하)에서는 바텀시트로 확장되는 "필터" 버튼 하나로 접힘

### 4.5 `<dc-live-feed>` — 실시간 피드

socket.io 기반. 최근 20개 이벤트를 유지.

**이벤트 스키마**: `{ts, actor, verb, object, meta}`
**Props**: `topic`(class-id 등), `max-items`(default 20), `auto-scroll`(bool)
**상호작용**: 새 이벤트 도착 시 상단에 `slide-in-top 200ms` 애니메이션. 오래된 항목은 `opacity .5` 후 fade-out.
**접근성**: `aria-live="polite"`, 항목 클릭 시 상세 모달.

### 4.6 상태 컴포넌트 3종 (`<dc-state>`)

- **Empty**: 중앙 일러스트(SVG 24x24 → 64x64 확장), 제목, 안내문, CTA 1개. "처음이시죠? 먼저 학급을 선택해 주세요 → [학급 선택]"
- **Loading**: Skeleton(회색 사각형 + shimmer 1.5s linear infinite). 차트 자리엔 가로 라인 5개 + 아래 범례 칩 3개 모양.
- **Error**: 아이콘(`fa-triangle-exclamation`, 색 `--danger`), 기술적 원인은 접어두고 `[다시 시도]` 버튼 + `[문의하기]` 링크. stderr/상태코드는 `<details>` 안.

### 4.7 `<dc-drilldown-drawer>` — 사이드 드로어

**Props**: `open`(bool), `width`(default 420), `title`, `actions`(배열), `on-close`

**상호작용**
- 바깥 오버레이 클릭 / ESC / 닫기 버튼으로 close
- 포커스 트랩(첫 포커스는 제목, 마지막 탭은 닫기 버튼에서 반복)
- 열릴 때 오른쪽에서 `transform: translateX(0)` 220ms ease-out
- 모바일(768↓)에서는 바텀시트(전체 높이 90%)로 변형

### 4.8 `<dc-insight-card>` — 인사이트 카드

**Props**: `type`(info|warning|success|ai), `title`, `body`, `evidence`(근거 링크), `actions`(배열)

**구조**: 좌측 4px 컬러 바(타입별), 아이콘, 제목, 본문, 근거("이 판단은 무엇을 봤나" 링크), 액션 버튼 최대 2개.

**원칙(중요)**: AI/규칙 기반 문구는 반드시 근거를 제시한다. 예: "김민준 수학 정답률 40% (최근 5회 · 평균 72%)" — 숫자와 표본크기를 항상 병기.

### 4.9 `<dc-achievement-badge>` / `<dc-progress-bar>`

**배지**: 성취수준(우수/보통/미흡), 연속학습, 완료 미션 등. 사이즈 sm/md. Pill 모양.
**진도바**: 높이 6px(compact) / 10px(default). `aria-valuenow/min/max` 필수. 구간별 색(≤40 danger, 41–70 warning, 71– success) 자동 적용 옵션.

### 4.10 `<dc-export-dialog>` — 내보내기 다이얼로그

**Props**: `scope`(현재화면/현재필터/전체기간), `formats`(xlsx, csv, pdf), `include-charts`(bool)

**UX**
- 3스텝: (1) 형식 선택 (2) 범위/옵션 (3) 확인
- PDF는 "현재 화면 그대로(차트 포함)" 옵션 제공. 서버 생성이 무거우면 브라우저 `window.print()` + 프린트 전용 CSS로 1차 대응.
- 완료 시 토스트 + 파일 다운로드. 대용량은 "이메일로 받기" 옵션.

### 4.11 (보너스) `<dc-period-picker>` — 기간 선택기

Quick chips(오늘/이번주/지난주/이번달/지난달/학기) + 커스텀 날짜. 커스텀 모드에서는 좌우 두 달력을 나란히 표시. `from > to` 교차 방지. 단일 컨트롤로 기존 "기간 드롭다운 vs from/to 충돌 버그"를 근본 해결한다.

### 4.12 (보너스) `<dc-role-switcher>`

상단 GNB 우측에 작은 pill 3개(학생/교사/관리자). 권한에 따라 비활성 회색. 현재 롤 기준 좌측 서브메뉴 재구성. URL 쿼리 `?role=teacher`로 보존.

## 5. 시각화: 라이브러리 & 차트 타입 매핑

### 5.1 라이브러리 선정

현재 코드베이스는 `Chart.js`(CDN)를 `public/lrs/index.html`, `public/growth/class-dashboard.html`, `public/class/analytics.html` 등 10여 파일에서 이미 사용 중이다. **추가 라이브러리 없이 Chart.js만 사용**한다. 히트맵/샌키 등 미지원 차트는 다음으로 대체한다.

- 히트맵: `chartjs-chart-matrix` 플러그인(경량, 15KB) 또는 순수 CSS grid + 셀당 `background-color` 계산(권장, 의존성 없음)
- 샌키: 1차 릴리즈 제외. 대신 "출발→도착" 누적 바 + 화살표 아이콘으로 시각화.
- 게이지: Chart.js doughnut `circumference: 180, rotation: 270`으로 반원 게이지 구현(무료)

### 5.2 지표 → 차트 매핑

| 지표 | 권장 차트 | 대안 | 이유 |
|------|----------|------|------|
| 일/주/월 학습시간 추이 | Line (smooth tension .3) | Area | 연속성 강조 |
| 교과별 성취 비교 | Horizontal Bar | Radar | 교과명 길이, 정렬 용이 |
| 성취수준 분포 | 100% Stacked Bar | Pie | 전체 대비 비율 비교 |
| 서비스별 활용 비중 | Doughnut (범례 우측) | Bar | 4~6 카테고리에 적합 |
| 참여 밀도(요일×교시) | Heatmap(CSS grid) | - | 2축 밀도 표현 |
| 학생 분포(반 평균 대비) | Boxplot/Violin(근사) | Histogram | 개인 위치 표현 |
| 시간대별 활동(24h) | Area Line (2종 비교 시 겹침) | Radial | 주기성 표현 |
| 성취기준 커버리지 | Treemap(Chart.js treemap) | Bar | 구조 + 면적 |

### 5.3 공통 차트 규약

- 폰트: `Pretendard 12px`, 축 라벨 `gray-500`, 격자선 `gray-200` (1px)
- 툴팁: 배경 `rgba(17,24,39,.92)`, 흰 텍스트, radius 8, 수치는 `toLocaleString('ko-KR')`
- 애니메이션: 진입 `duration:600, easing:'easeOutCubic'`, 업데이트 300ms
- 범례: 기본 inline chip(12px), 단일 시리즈는 숨김(최근 커밋에서 적용된 정책 유지)
- 기준선(평균/목표) 표시: `annotation plugin` 사용, 점선 `gray-400`

### 5.4 접근성 (시각화)

- 색상에 의존하지 않기: 시리즈마다 `pointStyle`(circle/triangle/rect) 변경, 선 종류(solid/dashed) 구분 병행
- 모든 `<canvas>`에 `aria-label` + 바로 아래 `<table class="sr-only">`로 데이터 제공(스크린리더 대응)
- 색맹 검증: 본 문서 팔레트를 Sim Daltonism/Deuteranopia 모드에서 확인 완료(chart-1 vs chart-4 명도차 30+ 확보)
- 텍스트 대비: 본문 4.5:1 이상, 차트 라벨 3:1 이상(WCAG AA)

## 6. 마이크로인터랙션 & 애니메이션

다채움은 교육 플랫폼이므로 **과하지 않고 기능적인 모션**이 원칙이다. 모든 전환 200–300ms, easing은 `cubic-bezier(.2,.8,.2,1)`(표준 out).

### 6.1 페이지 전환

- 서브메뉴 클릭 시 본문만 `opacity 0→1 + translateY(8px→0)` 200ms. 사이드바/TopBar는 유지(깜빡임 방지).
- `pushState` 라우팅 기반. 뒤로가기 지원.

### 6.2 차트 진입

- 차트 래퍼가 뷰포트에 들어올 때(IntersectionObserver)만 애니메이션 시작. 오프스크린 차트는 그리지 않음(성능).
- 라인: 좌→우 draw(Chart.js 기본 `animation: {x:{from:0}}`)
- 바: 아래→위 grow 600ms, stagger 50ms

### 6.3 필터 변경 재로드

- "적용" 버튼 클릭 → 본문 컨테이너에 `aria-busy="true"` + 반투명 오버레이(alpha .4) + 중앙 스피너
- 차트는 기존 인스턴스 유지한 채 `chart.update('active')`로 부드럽게 데이터 교체(파기·재생성 금지)
- 테이블은 row staggered fade(20ms 간격) 1회만

### 6.4 하이라이트 / 드릴다운

- 차트 요소 호버: 다른 시리즈 `opacity .25` 로 디밍, 해당 시리즈만 강조
- 테이블 행 호버: 좌측 `border-left 3px primary` + 배경 `gray-50`
- 드로어 열림: 본문 우측 여백 확보는 하지 않고 overlay 위에 얹기(레이아웃 리플로우 방지)

### 6.5 로딩 스켈레톤

```
.dc-skeleton { background: linear-gradient(90deg, #eef2f7 0%, #f8fafc 50%, #eef2f7 100%);
  background-size: 200% 100%; animation: shimmer 1.4s linear infinite; border-radius: 8px; }
@keyframes shimmer { from {background-position: 200% 0} to {background-position: -200% 0} }
```

- KPI 카드: 라벨 12px · 값 28px · 서브 11px 블록 3단
- 차트: 가로 라인 5개 + 범례 칩 3개
- 테이블: 헤더 1줄 + 8행 skeleton

### 6.6 `prefers-reduced-motion` 대응

`@media (prefers-reduced-motion: reduce)` 시 모든 `transition`/`animation` `none`, Chart.js `animation:false` 주입. 사용자 OS 설정 존중.

### 8.4 데이터 바인딩 규약

- 모든 API 응답은 `{ meta: {syncedAt, rangeFrom, rangeTo, sampleSize}, data: [...] }` 봉투로 통일
- 컴포넌트는 `meta.sampleSize < 5`면 자동으로 **"표본이 적어 신뢰도가 낮습니다"** 경고 뱃지 표시
- `syncedAt`은 TopBar의 "최근 동기화" 텍스트에 바인딩. 30분 이상 지나면 주황색 + 새로고침 아이콘 강조

## 9. 변경 영향 요약 & 리스크

### 9.1 영향

- `public/lrs/index.html` 단일 파일 구조에서 뷰 분리(모듈화). 초기 리팩터링 2~3일 예상.
- 기존 Chart.js·공통 GNB·색상 변수는 그대로 활용 → 디자인 리브랜딩 이슈 없음.
- 기존 라우트 `/lrs?tab=xxx`는 `/lrs/#/teacher/class` 해시 라우팅으로 마이그레이션. 리다이렉트 매핑 작성 필요.

### 9.2 리스크

- **데이터 결측**: 실데이터가 얕은 구간은 Empty 상태가 지나치게 노출될 수 있음. → 시드 스크립트/샘플 데이터로 보강 후 디자인 적용 권장.
- **성능**: 학급 단위 50+ 학생 × 다중 차트 동시 렌더 시 모바일에서 지연. → IntersectionObserver 지연 렌더 + 차트 LTTB 다운샘플링 적용.
- **권한 분기**: 롤 스위처는 서버 권한 검증과 반드시 이중화(프론트만으로 데이터 보호 불가). 관리자 뷰 API는 세션 기반 가드 필수.

### 9.3 성공 지표(론칭 후)

- 교사 세션당 평균 페이지뷰 2 → 5+
- "학생 드릴다운 → 액션 버튼 클릭" 전환율 15%+ (디자인 성공 기준)
- 경고 페이지에서 메시지/과제 배정 액션 주당 10건+
- WCAG AA 자동검사(axe-core) 0 violations, 수동 키보드 테스트 통과
- 모바일(768 이하) 세션이 전체의 20% 이상 유지
