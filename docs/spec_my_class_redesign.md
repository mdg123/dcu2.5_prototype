## 1. 목적과 배경

### 1.1 현재 상황의 문제점 (실측 기반)

- **문제 1 — 학생/학부모 첫 화면이 빈 화면**: 기본 탭이 "개설 클래스"라 학생·학부모는 개설한 클래스가 없어서 빈 폴더 아이콘만 보게 된다. 진입 가치 -100점.
- **문제 2 — 한 사용자가 두 입장 동시 보유 시 탭 왔다갔다**: 교사가 자기 반(개설)과 동학년 협의회(참여) 둘 다 가지면, 매번 탭 전환해야 한다. 통합으로 한 번에 보고 싶음.
- **문제 3 — "전체 수업 25개" 같은 무의미한 통계**: 통합 관점에서 나에게 의미 있는 숫자는 "내가 응시 안 한 평가" 같은 행동 가능 숫자다.
- **문제 4 — 개설자/멤버 시각 구분 부재**: 카드만 보면 누가 만든 클래스인지 알 수 없다.
- **문제 5 — 모바일에서 헤더 버튼이 본문을 압도**: "코드로 가입 / 클래스 만들기" 두 버튼이 페이지 제목 옆에 나란히 있어 모바일 375px에서 페이지 제목이 두 줄로 잘림.
- **문제 6 — 5원칙 미충족**: (b) 현재 위치 피드백이 "탭 활성"뿐 + (d) 빈 상태가 "개설한 클래스가 없습니다" 한 줄 텍스트로 끝남.

### 1.2 통합 목표 (Goal)

> "어느 역할이든 처음 들어와서 1초 만에 **'오늘 무엇을 해야 하는지'**, 5초 만에 **'어느 클래스에서 무엇이 일어났는지'** 파악한다."

### 1.3 사용자 5원칙 적용 포인트

| 원칙 | 적용 |
|------|------|
| (a) 진입 경로 | 상단 "오늘 할 일" 배너에서 우선 행동을 즉시 안내. 클래스 카드는 클릭 한 번으로 클래스 홈으로 이동 |
| (b) 현재 위치 피드백 | 필터칩(전체/개설/참여) active 상태 강조. 카드 hover 시 좌측 라인 색 변경. 캘린더 선택일 강조 |
| (c) 복귀 동선 | 모든 모달·사이드바 상단 우측 × 버튼 + Esc 키 닫기. 캘린더 사이드바는 모바일 전체 화면 차지 시 좌상단 ‹ 백 화살표 |
| (d) 빈 상태·오류 상태 | 클래스 0개 시 "첫 클래스를 만들어 볼까요?" + CTA 두 개(만들기 / 코드 가입). API 오류 시 "잠시 후 다시 시도해 주세요" + 재시도 버튼 |
| (e) 평이한 한국어 | "활동 내역" → "내 할 일 / 처리 대기" 같은 행동 동사형. "응시" 같은 한자 단어는 그대로 두되 "(시험 보기)" 부연 |

| 역할 × 입장 | 가장 보고 싶은 정보 (우선순위 1→4) | 카드 표시 차별점 | 4슬롯 통계 |

| **학생 × 멤버** (기본) | ① 오늘 마감 과제 D-Day ② 미응시 평가 ③ 새 공지 ④ 다음 수업 시간표 | 🎓 멤버, 미완료 작업 카운터(`📋1 ⭐1`) | ① 오늘 마감 ② 미응시 평가 ③ 새 알림 ④ 미확인 콘텐츠 |

| **학부모 × 멤버** (기본) | ① 자녀 마감 임박 과제 ② 미확인 콘텐츠 ③ 새 알림 ④ 자녀 출결 상태 | 🎓 멤버, **자녀 이름 라벨**(`👶 이학생`), 미확인 콘텐츠 카운터 | ① 미확인 콘텐츠 ② 마감 임박 과제 ③ 새 알림 ④ 자녀 출결 |
| **학부모 × 개설** (학부모회 등) | ① 새 게시글 ② 새 멤버 ③ 오늘 일정 ④ 미답변 | 👑 개설 + 학부모회용 회색 톤 | 학부모 멤버와 동일 + "새 게시글" 슬롯 |
| **교직원 × 개설** | ① 미확인 공지 ② 이번 주 학사 일정 ③ 멤버 수 ④ 안내 완료율 | 👑 개설, 🏛 행정 배지(슬레이트), 안내 완료율 카운터 | ① 미확인 공지 ② 이번 주 학사 일정 ③ 멤버 수 ④ 안내 완료율 |
| **교직원 × 멤버** | ① 새 행정 공지 ② 결재 대기 ③ 이번 주 일정 ④ 미확인 게시글 | 🎓 멤버 + 🏛 행정 배지 | 교직원 개설과 동일 |

> **공통 표준화 가능 영역**: 카드의 "썸네일 + 클래스명 + 멤버 수 + D-Day 칩"은 모든 페르소나 공통. 차별화 영역은 "카운터 칩"과 "4슬롯 통계 라벨"뿐. → 컴포넌트 1개로 가능, 데이터 슬롯만 역할별 다르게 주입.

## 7. 반응형 분기

### 7.1 데스크톱 (≥1200px) — 기준 레이아웃

- 메인 그리드: 좌(클래스 목록) 50% / 우(캘린더) 50% — 현재 유지
- 4슬롯 통계: 4컬럼 grid (현재 유지)
- 콘텐츠 섹션 (최근 수업/과제/알림): 3컬럼 grid (현재 유지)
- 활동 내역 카드: progress bar + 액션 버튼 우측

### 7.2 태블릿 (768~1199px)

- 메인 그리드: 1컬럼 — 클래스 목록 → 캘린더 순서
- 4슬롯 통계: 2×2 grid

## 8. 데이터 흐름 + API 매핑

### 8.1 진입 시 호출되는 API

| 순서 | API | 목적 | 비고 |
|------|-----|------|------|
| 1 | `GET /api/auth/me` (common-nav.js 자동) | 사용자 role 식별 (student/teacher/parent/staff/admin) | 역할별 분기 모든 부분의 트리거 |
| 2 | `GET /api/class` | 내 클래스 목록 | `my_role` 필드로 개설/멤버 구분 |
| 3 | `GET /api/class/new-counts` | 클래스별 새글 종류별 카운트 | 이미 lessons/homework/exams/notices/posts 별로 분리되어 옴 → 그대로 칩에 매핑 |
| 4 | 각 클래스별 `/api/lesson/{classId}`, `/api/homework/{classId}`, `/api/exam/{classId}`, `/api/notice/{classId}` (병렬) | 통계·캘린더·활동 내역용 | 현재 코드 유지 |
| 5 | `GET /api/portal/calendar?startDate=&endDate=` | 캘린더 이벤트 | 현재 코드 유지 |

### 8.2 신규 필요 데이터 (백엔드 추가 필드 또는 신규 엔드포인트)

| 항목 | 현재 상태 | 필요한 추가 |
|------|-----------|------------|
| 핀 고정 클래스 | 없음 | localStorage(우선) → 추후 `pinned` boolean 컬럼 (`class_members` 테이블) |
| 학부모-자녀 매핑 | `users.parent_id` 컬럼 존재 | **신규 GET `/api/class/children-mapping`** — 학부모 사용자에게 자녀 ID·이름·소속 클래스 ID 반환 |
| 미채점 과제 수 | 없음 (각 homework에 응답 카운터는 있음) | **신규 GET `/api/class/{classId}/owner-summary`** — 미채점, 미제출, 오늘 수업, 마감 임박 평가를 한 번에 |
| 안내 완료율 (교직원) | 없음 | **신규 GET `/api/class/{classId}/staff-summary`** — 공지 응답률, 학사 일정, 응답 대기 |
| "오늘 할 일" 통합 배너 데이터 | 없음 | **신규 GET `/api/class/today-summary?role={role}`** — 역할별 분기된 오늘 할 일 합계 (선택: 클라이언트에서 합산 가능, 단 4-5개 API가 합쳐 호출되므로 서버 합산 권장) |
| 마감 임박 (D-3 이내) 항목 | 클라이언트에서 계산 | 그대로 유지 — 클라이언트 산출 |

### 8.4 데이터 → UI 매핑

```
my_role (DB) ─────┬─→ 카드 우상단 배지(👑/🎓)
                  └─→ 필터칩 [개설/참여] 분류
class_type ───────→ 카드 메타 ("학년/반" 등) + 그룹핑 옵션 키
new-counts.lessons → 카운터 칩 📚N (안 쓰는 경우 생략. 우선 📣📋⭐💬에 매핑)
new-counts.notices → 칩 📣N (공지)
new-counts.homework→ 칩 📋N (과제)
new-counts.exams ──→ 칩 ⭐N (평가)
new-counts.posts ──→ 칩 💬N (게시글)
homework.due_date  → D-Day 칩 (계산: ceil((due - today) / 86400000))
homework.submission_count + cls.student_count → 카드 우측 "👥 미제출 N명" (개설자만)
```

## 10. 추가 권고 (구현 시 함께 고려)

### 10.1 단계별 출시 (구현 분리 권장)

> 한 번에 다 바꾸면 검증 부담이 큼. 다음 순서로 분리:

| Phase | 범위 | 예상 변경 파일 |
|-------|------|--------------|
| P1 | 탭 → 필터칩 + 왕관/모자 배지 + 새글 종류별 칩 | `public/class/index.html` (HTML+CSS+JS) |
| P2 | "오늘 할 일" 배너 + 4슬롯 라벨 역할별 분기 | + 신규 API `/api/class/today-summary` (`routes/class.js`, `db/class.js`) |
| P3 | 학부모 자녀 그룹핑 + 자녀 라벨 | + 신규 API `/api/class/children-mapping` |
| P4 | 핀 고정 + 그룹핑 토글 + 모바일 미니 캘린더 | localStorage + CSS 위주 |
| P5 | 빈/오류 상태 메시지 정비 + 접근성(aria-label, 키보드) | 디테일 마감 |

### 10.2 접근성 (도메인 전문가 권고 사항)

- 모든 카드에 `role="link"` + `aria-label="{클래스명} 클래스로 이동"` 부여
- 필터칩에 `role="tablist"` 대신 **`role="group"` + 각 칩에 `aria-pressed`** (탭이 아닌 다중 필터처럼 동작 가능하도록)
- 4슬롯 통계 클릭 시 `aria-describedby`로 어떤 페이지로 가는지 명시
- 색맹 대응: D-Day 칩은 색뿐 아니라 ⏰ 아이콘으로도 구분 (현재 명시됨)
- 키보드 내비게이션: 카드 Tab 순서 + Enter로 진입

### 10.3 미래 확장 여지

- "이 클래스에서 나는?" 미니 위젯 (개설자 + 멤버 동시 보유 시 카드 안에 토글)
- 카드 우상단 ⋯ 메뉴에 "개설자에게 쪽지" (멤버 입장)
- 학부모 자녀 2명 이상일 때 "오늘 할 일" 배너에 자녀별 탭
