**작성**: 정책기획자/QA 테스터 서브에이전트
**작성일**: 2026-04-21
**대상**: `public/lrs/index.html` (LRS 학습분석 대시보드) 및 연관 백엔드(`routes/lrs.js`)
**목적**: 교육부/시도교육청 "다차원 학생성장 플랫폼 품질 제고 사업"의 정책 목표를 본 프로토타입이 충족하는지 검증하고, Phase 4 QA에 사용할 구체적 테스트 기준을 정의한다.

## 1. 정책 배경

### 1.1 교육부 AIDT 및 LRS 학습분석 정책 방향

2025년 3월부터 단계적으로 도입된 **AI 디지털교과서(AIDT)** 정책은 모든 학생의 학습 데이터를 실시간으로 수집·분석하여 **개별 맞춤형 학습**을 지원하도록 의무화하고 있다. 이의 기술적 기반이 **LRS(Learning Record Store)** 이며, xAPI(Experience API) 표준을 채택하여 학습 활동을 "행위자-동사-객체(Actor-Verb-Object)" 구조로 저장하도록 권고한다. 시도교육청은 각 지역 LRS를 구축하여 교육부 K-에듀파인 허브와 연계해야 한다.

충청북도교육청의 **다채움 플랫폼**은 기존 채움클래스·채움콘텐츠·스스로채움·우리반 성장기록·채움CBT 5개 서비스를 통합한 LRS로서, 다음 5가지 정책 목표를 달성해야 한다.

### 1.2 핵심 정책 목표

1. **2022 개정 교육과정 성취기준 연계**: 모든 학습 활동 로그에 성취기준 코드(예: `[6수03-05]`)를 부여하고, 학생별 성취기준 도달도를 집계·시각화한다.
2. **개별 맞춤형 학습 지원 의무**: 학생의 강점/약점 영역을 자동 식별하고, 부진 영역에 대한 보충 학습 경로를 제시한다.
3. **교사 수업 의사결정 지원**: 교사가 반 전체 도달도와 개별 학생 부진 경보를 한눈에 파악하고, 차시 수업을 조정할 근거를 제공한다.
4. **학부모 공유 투명성**: 학부모가 자녀의 학습 이력·성취 현황을 이해하기 쉬운 형태로 확인할 수 있도록 리포트를 제공한다.
5. **개인정보 보호 및 데이터 주권**: 「개인정보 보호법」·「교육기본법」 및 교육부 가이드라인에 따라 최소수집·목적제한·접근통제 원칙을 준수한다.

### 1.3 법·제도적 요구

- **개인정보 보호법 제23조** — 민감정보(학업성취도 포함) 별도 동의·암호화 저장
- **교육부 훈령** — 학적·출결 정보는 7년 보존, 학습활동 로그는 3년 보존 권고
- **시도교육청 AIDT 운영 지침** — 교사·학생·학부모 역할별 접근권한 분리, 학교 외부 반출 금지
- **WCAG 2.1 AA** — 공공교육 플랫폼 접근성 의무 준수

## 3. 사용자 시나리오 20개

### 학생 역할 (S-01 ~ S-05)

**S-01. 오늘 내 학습 확인**
- 역할: 학생 | 상황: 방과 후 집에서 오늘 공부한 내용 확인
- 목표: 오늘 푼 문제 수·정답률 확인
- 흐름: 로그인 → LRS 홈 → "내 학습" → 오늘 날짜 필터
- 기대: 오늘 활동 건수·학습시간·정답률 카드 표시
- AC: 데이터가 있으면 숫자 표시, 없으면 "오늘 학습 기록이 없어요" 빈 상태

**S-02. 취약 단원 파악**
- 학생 | 단원평가 전 복습 | 내 약한 단원 찾기
- 흐름: 학업성취 → 성취기준별 현황 → 도달도 낮은 순 정렬
- 기대: 60% 미만 성취기준 목록 + 관련 학습콘텐츠 링크
- AC: 3개 이상 취약 단원 있을 때 상위 3개 강조 표시

**S-03. 지난주 학습 시간 추이**
- 학생 | 주간 리뷰 | 내가 얼마나 꾸준했는지 확인
- 흐름: 학습현황 → 학습활동 시간 → 최근 7일
- 기대: 요일별 학습시간 바 차트, 평균선 표시
- AC: 빈 요일은 0으로 표시, 툴팁에 분 단위 시간

**S-04. 오답노트 이력 확인**
- 학생 | 자기주도학습 | 내가 틀린 문제 다시 풀었는지 확인
- 흐름: 학습수행 → 콘텐츠 활용 → 오답노트 필터
- 기대: 재시도 횟수·정답 여부 타임라인
- AC: 각 문항 클릭 시 원 문항 미리보기

**S-05. 나의 성장 리포트 공유**
- 학생 | 학기말 | 학부모에게 보여줄 요약
- 흐름: 홈 → "리포트 내보내기" → PDF/이미지 저장
- 기대: 1페이지 요약(총 학습시간·주요 성취·성장 그래프)
- AC: 파일명에 학생명·기간 포함, 민감정보 필터링

### 교사 역할 (T-01 ~ T-08)

**T-01. 1교시 수업 전 반 전체 진도 체크**
- 교사 | 수업 직전 | 학생들이 예습 과제를 했는지 확인
- 흐름: 교사 대시보드 → 반 선택 → 과제 제출률
- 기대: 제출 완료/미완료 학생 수 + 미제출 명단
- AC: 미제출 ≥20% 시 경고 배너 표시

**T-02. 부진학생 조기 발견**
- 교사 | 주 1회 점검 | 도달도 낮은 학생 식별
- 흐름: 학업성취 → 학생별 도달도 → 경보 학생 탭
- 기대: 도달도 <60% 학생 카드 + 부진 교과/단원 표시
- AC: 경보 학생에게 1:1 메시지 보내기 버튼 노출

**T-03. 교과별 난이도 진단**
- 교사 | 단원평가 후 | 어떤 문항이 어려웠는지 확인
- 흐름: 학업성취 → 평가문항 분석 → 정답률 낮은 순
- 기대: 문항별 정답률 + 오답 유형 상위 3개
- AC: 정답률 30% 미만 문항 빨간 배지

**T-04. 콘텐츠 활용 효과 분석**
- 교사 | 수업 자료 선택 시 | 어떤 콘텐츠가 효과적이었나
- 흐름: 학습수행 → 콘텐츠 활용 → 완료율·학습시간 정렬
- 기대: 콘텐츠별 조회/완료/평균시간 표
- AC: CSV 내보내기 가능

**T-05. 개별 학생 드릴다운**
- 교사 | 학부모 상담 준비 | 특정 학생 전체 기록
- 흐름: 학생 목록 → 학생명 클릭 → 개인 리포트
- 기대: 최근 30일 타임라인, 교과별 도달도, 학습 패턴
- AC: 교사는 담임 반 학생만 열람, 타 반 접근 시 403

**T-06. 교사 본인 활용도 자가진단**
- 교사 | 장학 대비 | 내가 플랫폼을 얼마나 활용했나
- 흐름: 교사활용지수 메뉴
- 기대: 콘텐츠 업로드·과제 출제·피드백 작성 횟수
- AC: 동료 교사 평균 대비 본인 퍼센타일

**T-07. 맞춤 학습 과제 자동 생성**
- 교사 | 주간 과제 | 학생별로 다른 과제 주고 싶음
- 흐름: 맞춤학습 → 학생 선택 → "취약 단원 기반 과제 생성"
- 기대: 학생별 취약 단원 문항 5개 자동 선별
- AC: 생성된 과제를 채움클래스로 전송 가능 (Should Have)

**T-08. 일일 현황 브리핑**
- 교사 | 아침 | 전날 반 전체 활동 요약
- 흐름: 일일현황 → 어제 날짜
- 기대: 총 학습시간·활동건수·미접속 학생 수
- AC: 미접속 3일 이상 학생 명단 표시

### 학부모 역할 (P-01 ~ P-03)

**P-01. 자녀 이번 주 학습 확인**
- 학부모 | 주말 | 자녀가 공부를 얼마나 했나
- 흐름: 포털 로그인 → 자녀 선택 → LRS 학부모 뷰
- 기대: 주간 학습시간·완료 과제 수·주요 활동 3건
- AC: 상세 점수는 마스킹, 성취 레벨(A~E)만 표시

**P-02. 학기말 성장 리포트 수신**
- 학부모 | 학기말 | 자녀 성장 총평
- 흐름: 알림 → 리포트 다운로드
- 기대: PDF 1~2페이지, 강점/약점/담임 코멘트
- AC: 자녀 본인·부모만 접근 가능

**P-03. 취약점 알림 수신**
- 학부모 | 수시 | 자녀가 어려워하는 부분 파악
- 흐름: 알림 센터 → "수학 분수 단원 도달도 낮음" 알림 클릭
- 기대: 보충 학습 콘텐츠 추천
- AC: 알림은 부모 동의한 경우에만 발송

### 관리자 역할 (A-01 ~ A-04)

**A-01. 학교 전체 활용 현황 모니터링**
- 관리자 | 월간 보고 | 학교 플랫폼 활용도
- 흐름: 관리자 대시보드 → LRS 통계
- 기대: 학급별 활성 사용자·일평균 학습시간
- AC: 학생 개인정보 비노출, 집계값만

**A-02. 교사 활용 격차 확인**
- 관리자 | 연수 계획 | 저활용 교사 파악
- 흐름: 교사활용지수 → 학교 전체 분포
- 기대: 히스토그램 + 하위 20% 익명 카운트
- AC: 교사 실명 비노출(관리자라도 집계만)

**A-03. 데이터 품질 점검**
- 관리자 | 분기 점검 | 로그 누락·오류 확인
- 흐름: 관리자 → 데이터 품질 리포트
- 기대: 서비스별 로그 수집 정상률, 이상치 건수
- AC: 정상률 95% 미만 시 경고

**A-04. 개인정보 접근 감사**
- 관리자 | 감사 대응 | 누가 누구 데이터를 언제 봤나
- 흐름: 관리자 → 접근 로그
- 기대: 교사 ID·조회 대상·시각 로그
- AC: 최소 1년 보관, CSV 내보내기

## 7. 결론 및 권고

본 프로토타입은 충청북도교육청 품질 제고 사업의 **"실데이터 기반 학습분석 대시보드"** 요구를 충족하기 위해 최소 Must Have 10개 항목과 P0 테스트 케이스 15건을 반드시 통과해야 한다. 특히 다음 3가지를 최우선으로 해결할 것을 권고한다.

1. **스키마 마이그레이션 (R1)** — 모든 후속 작업의 전제. Phase 2 착수 전 완료.
2. **현실적 시드 데이터 (R2)** — 공백 UI가 완성도 인상을 크게 해침. 학생 30명·30일치 로그 필수.
3. **역할별 권한 분리 (M5, M6, R4)** — 정책·법적 리스크로 직결. 테스트 TC-031~038 전수 통과 필수.

Should Have 항목(AI 추천, 학부모 리포트 PDF 등)은 Must Have 완료 후 주차별 1~2개씩 점증적으로 추가하되, 일정 압박 시 **"프로토타입 범위 외, 향후 본사업 구현"** 으로 명시적 배제하여 완성도를 우선한다.

본 문서는 Phase 4(통합 검토 라운드 1)에서 체크리스트로 활용하며, 발견 이슈는 별도 이슈 트래커(`docs/qa-audit-report.md`)에 연동 기록한다.
