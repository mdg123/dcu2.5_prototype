## 0. 사용자 강조 사항(요약)

> "대시보드가 제대로 동작하지 않는 것 같음."

실측 결과 — **사용자 지적은 사실이며, 시각·기능 양쪽이 모두 망가져 있음.**
- `/api/self-learn/dashboard` 응답 키와 `loadDashboard()`가 사용하는 키가 어긋나 4개 카드 중 **2개(총 학습 시간·랭킹)는 데이터 소스가 아예 없음**.
- 같은 페이지의 "나의 기록" 탭은 19건/37% 등 정상 수치를 보여 주지만, 상단 4-grid 카드는 0/0%/0분/- 로 표시되어 **한 화면에 모순된 두 값이 공존**한다.
- "랭킹" 탭은 모든 학생이 0점(샘플 데이터 미주입)으로 표기되어 카드와 탭 모두 의미를 잃음.

## 1. Critical (즉시 수정)

### [UI-C-001] 대시보드 4-grid 카드 — 데이터 결손 / 동일 화면 수치 충돌
- 화면: AI 맞춤학습 상단 (learning-map.html L650~683)
- 캡처: 본 보고서 §부록 그림 1
- 실측:
  - `dashCompleted`: API `completedNodes` 사용 → `0` (정상 키 매칭)
  - `dashProgress`: API `progressPercent` 또는 `completed/total` → `0%` (정상)
  - `dashTime` (총 학습 시간): API에 `total_time_minutes`/`totalTimeMinutes` 키가 **존재하지 않음** → 항상 `0분`
  - `dashRank` (랭킹): API에 `rank` 키가 **존재하지 않음** → 항상 `-`
  - 동일 페이지 "나의 기록" 탭은 `total_solved=19, avg_accuracy=37%`를 정상 출력 → 사용자가 **0과 19를 같은 화면에서 동시에** 보게 됨
- 영향: 사용자가 시스템을 신뢰할 수 없음. CLAUDE.md "(d) 빈/오류 상태 친절한 안내" 위반.
- 권장: API 응답에 `total_time_minutes`, `rank`, `rank_total` 추가 + `loadDashboard()`가 `dashboardCache`와 "나의 기록" 탭이 동일 필드를 사용하도록 단일화. 데이터가 비어 있으면 "기록 없음 / 학습을 시작해 보세요" 안내로 대체.
- 스펙: 카드 padding 16px 18px(현행 유지), value font-size 24px → **28px**(단일 강조 수치 가독성), label 13px / sub 12px / value color `#111827`, sub color `#6B7280`. 빈 상태 value `--`(em-dash) + sub "데이터가 쌓이면 표시됩니다".
- 영향 파일:
  - `public/self-learn/learning-map.html` line 1379–1393 (`loadDashboard`)
  - `routes/self-learn` 또는 dashboard API 응답 핸들러

### [UI-C-002] 진단 모달 헤더/바디 카운터 불일치
- 화면: 진단평가 → 진단하기 모달 (diagModal)
- 캡처: §부록 그림 4
- 실측: 헤더 좌상단 "풀이 1/3 · 문항 1/2", 바디 헤더 "문항 1/1 · 객관식 10점" — 같은 모달에서 분모가 1, 2, 3 세 가지가 동시에 보임
- 권장: 단일 진행 표시로 통일 — 헤더에는 "진단 1/3 단계", 바디 상단에는 "문항 N/총문항수 · 배점"만. 헤더의 "풀이 X/3"은 단계(인트로/풀이/결과)로, "문항 X/Y"는 현재 단계 내 진행으로 분리해 라벨링.
- 스펙: 헤더 진행 배지 13px / `#4B5563` / 배경 `#F3F4F6`. 바디 진행 배지 13px / 색상 동일.

### [UI-C-003] 진단 모달 닫기(×) 버튼과 진행 배지 인접 — 40px 룰 부근 위반 가능
- 화면: diagModal 우상단
- 실측: 캡처 기준 헤더 우측 × 버튼과 좌측 진행 배지 사이 가로 거리는 화면 폭에 의존, × 버튼 자체는 우상단 단독이라 직접 겹침은 없음. 단, 헤더 좌측에 진행 텍스트가 길어지면(예: "풀이 11/12 · 문항 11/12") × 와의 시각 균형이 깨짐.
- 권장: × 버튼 영역을 32px 정사각 + 우/상단 padding 16px 확보, 헤더 좌측 텍스트는 max-width: calc(100% - 64px) ellipsis.

### [UI-C-004] 페르소나 자동 전환(학생→교사) 버그 — 동선 단절
- 화면: `/self-learn/today.html` 진입 시 헤더가 "교사 김선생"으로 자동 전환됨(원래 학생 화면이어야 함)
- 캡처: §부록 그림 5
- 영향: 학생 시나리오 "오늘의 학습 → AI 맞춤학습 → 오답노트" 동선 자체가 무너짐. 디자인 점검 이전에 페르소나 라우팅이 깨져 있음.
- 권장: dev 환경 자동 로그인 fixture 검증 필요. UI 측면에서는 헤더 우상단 페르소나 토글이 **현재 페르소나를 색상·아이콘으로 명확히** 보여 주어야 함(현재는 작은 텍스트만).

## 2. High

### [UI-H-101] "학습 랭킹" 탭 전체 0점 — 빈 상태 미정의
- 화면: AI 맞춤학습 → 학습 랭킹 (§부록 그림 3)
- 실측: 8명 학생 모두 "0점", 1·2·3위 메달 아이콘만 색상 차이(노/회/구). 점수가 모두 같으면 순위 산정 의미가 사라짐.
- 권장: "아직 점수 기록이 없어요. 진단평가나 차시 학습을 완료하면 점수가 쌓입니다." 빈 상태 카드로 대체. 적어도 한 명 이상 점수 > 0일 때만 리스트 렌더.
- 스펙: 빈 상태 카드 padding 48px / 일러스트(트로피 아이콘 64px) / title 18px / desc 15px / `#6B7280`.

### [UI-H-102] 노드 카드 가독성 — 폭 121px / 높이 72px에 5줄 텍스트
- 화면: 학습맵 SVG 노드 (status-available 등)
- 실측: 121×72px 안에 "초1학년 / 수와 연산·1학기 / 한 자리 수의 덧셈과 뺄셈 / 📚 12개 차시 / 학습가능" 5줄. 제목 길어지면 자동 줄바꿈으로 카드 박스 깨짐 위험.
- 권장: 카드 최소 폭 140px, 제목 2줄 ellipsis(`-webkit-line-clamp:2`), 학년·영역·학기는 한 줄로 합치고 폰트 11px / 색 `#6B7280`. 제목 13px / `#111827` / weight 600. "학습가능" 등 상태 라벨은 카드 하단 컬러바(2px) + 우측 도트(8px)로 대체.
- 영향 파일: learning-map.html `.svg-node` 스타일

### [UI-H-103] 4-grid 카드 value 폰트 24px — 공통 스케일과 어긋남
- CLAUDE.md 공통 스케일: body 17 / h1 28-30 / button 16 / badge 13. 24px는 정의되지 않은 중간값.
- 권장: 28px(h1과 동일) 또는 22px(보조 수치) 중 의도에 맞춰 한 가지로 통일. 본 카드는 핵심 KPI이므로 28px 권장.

### [UI-H-104] 노드 드로어 헤더 백버튼+브레드크럼+배지+타이틀 4단 → 세로 공간 과다
- 화면: nodeDrawer header (캡처 그림 2)
- 실측: 백버튼(차시 목록으로) / 브레드크럼 / 배지(수와 연산·초1학년·학습가능) / 타이틀 / 부제 — 5줄. 컴팩트화 필요.
- 권장: 1줄째 백버튼+브레드크럼, 2줄째 배지(인라인 3개), 3줄째 타이틀(20px) + 부제(13px) 같은 줄. 총 3줄로 축약. 차시 목록 카드와 헤더의 비율 80:20 유지.

### [UI-H-105] "이 단원의 차시" 카드 진행률 바 0% × 5개 — 색 정보 없음
- 캡처 그림 2: 5개 차시 모두 0% 회색 바. 기본 회색 한 가지만으로는 "학습 중"·"완료"·"새 차시" 구분 불가(현재 한 차시만 노란 "학습 중" 배지).
- 권장: 진행률 바 색상 — 0% 회색 `#E5E7EB`, 1~99% 파랑 `#3B82F6`, 100% 초록 `#10B981`. 차시 카드 좌측 4px 컬러 스트라이프(상태별).

## 3. Medium

### [UI-M-201] 단원 드로어 푸터 3-버튼 동일 너비 — 강조 CTA 분실
- 화면: nodeDrawer 푸터 — [진단하기] [첫 차시 바로 학습] [학습목록 추가]
- 실측: 캡처에서 노란/연파/연보 3색 동일 사이즈 → 한 화면 강한 컬러 1개 룰 위반.
- 권장: 메인 CTA 1개(첫 차시 바로 학습 — 파랑 `#3B82F6` solid) / 보조 1개(진단하기 — outline 파랑) / 텍스트 버튼 1개(학습목록 추가 — link). 사용자가 "지금 무엇을 누르면 되는가" 한 번에 보이도록.

### [UI-M-202] 학습맵 필터 바 가로 스크롤 가능성
- 실측: 학년 칩 6개(초1~초6) + 영역 칩 4개 + 줌 컨트롤 + 우측 +/− 66% 핏 — 1280 이하에서 잘릴 위험.
- 권장: 학년/영역을 별도 줄로 분리, 줌은 캔버스 우상단 floating(현 상태 일부 그러함) 유지. 필터 칩 좌측 라벨 "학년" "영역" 12px 회색.

### [UI-M-203] 빈 상태 — "학습 경로" 탭 일러스트·메시지 약함
- 캡처 그림 3: "아직 학습 경로가 없습니다 / 진단평가를 완료하거나 AI 경로 생성 버튼을 눌러 경로를 만들어 보세요." — 메시지는 적절. 그러나 다음 액션 버튼이 우상단 "AI 경로 생성"에 있어 빈 상태 카드 안에서 클릭 유도가 약함.
- 권장: 빈 상태 카드 안에 보조 버튼 [AI 경로 생성] 복제 + 보조 [진단평가 시작]. 카드 padding 64px, 일러스트 80px.

### [UI-M-204] "오늘의 학습" 진행 카드 — Lv.2 호기심 새싹 상단 비어 있음
- index.html 학생 홈 캡처(그림 5'): 좌측 도넛 0%, 가운데 "오늘의 학습 0/0개 · AI 맞춤학습 0/146 · 오답노트 정리 0/29" 우측 캐릭터. 0/0은 사용자에게 "할 일이 없는지/배정이 안 된 건지" 모호함.
- 권장: "오늘 배정된 학습이 없어요" 명시 + "AI 맞춤학습 시작" 버튼 인라인 노출.

### [UI-M-205] 차시 라벨 너무 짧음(`📚 5개 차시`) vs 실제 5개 미만/없음 케이스 미설계
- 권장: `📚 차시 5개` 한국어 어순 + 0개일 때 `차시 준비 중` 라벨로 변경.

### z-index / 모달
- diagModal `z-index: 9999` 추정(스타일 직접 미확인). CLAUDE.md ≥10000 권장 미달 가능 → 11000 권장.

### 반응형
- 1280 desktop 기준 카드 388px×4 정상. 1024 tablet 시 4→2 전환 미설정 추정. 권장: `grid-template-columns: repeat(auto-fit, minmax(260px, 1fr))`.
