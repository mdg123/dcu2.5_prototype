## 0. 재기획 핵심 가설

> **"매핑은 검색이 아니라 매칭 작업이다."**
> 1,162개 노드 × 평균 2~3개 콘텐츠 = 약 3,000~3,500건의 매핑을, **AI 추천을 기본 채널로** 삼고 **사람은 검수만** 하면 운영 비용이 1/10로 떨어진다.
> 동시에 처음 진입한 관리자도 30초 안에 첫 매핑을 완료할 수 있어야 한다.

설계 원칙:
1. 화면을 **3-pane 워크스페이스**로 확장 (모달 비의존).
2. **AI 추천을 1차 채널**로, 검색은 2차, ID 직접 입력은 3차로 강등.
3. **매핑 통계 대시보드**를 상단에 띄워 진행률·고립 노드를 한눈에 노출.
4. **CSV 가져오기/내보내기**로 외부 일괄 작업 허용.
5. 빈/오류/경계 상태에 **CTA 또는 재시도** 부착.

### 안 1. 매핑 워크스페이스 신설 (3-pane 레이아웃)

```
┌──────────────────────────────────────────────────────────────────────────┐
│ [학습맵 관리 > 노드-콘텐츠 매핑]   매핑률 38% (442/1,162)   [CSV ↓][CSV ↑] │
├──────────────────────────────────────────────────────────────────────────┤
│ [통계 칩 4개: 전체 38% · 영상 매핑 312 · 문항 매핑 580 · 고립노드 720]   │
├──────────────────────────────────────────────────────────────────────────┤
│ ┌─ 좌 노드 트리 320px ─┐┌─ 중 매핑 상세 1fr ────────┐┌─ 우 콘텐츠 패널 380px ─┐

│ │       · 2차시 ●○○ ←  ││  │ 영상 #1 [↑↓ ✕]  │    ││  │ 정확도 92% [추가] │  │

│ │ ☐ 미매핑만 보기      ││  + 콘텐츠 추가 (또는    ││  검색 입력 [🔍]        │

```

#### 좌측 — 노드 트리 (320px 고정)
- **트리 형태로 학년 → 단원 → 차시** drill-down. 각 노드 옆에 매핑 비율 표시:
  - 단원: `(매핑된차시 / 총차시)` 텍스트
  - 차시: `●●○` 3개 도트 — 영상유무/문항유무/추천검수완료 3축
- **미매핑 노드**: `border: 1px dashed #d1d5db; background: #f9fafb; color: #9ca3af;` 회색 점선
- **부분 매핑**: `border: 1px solid #fde68a; background: #fffbeb;` 주황 약하게
- **완료 매핑**: `border: 1px solid #a7f3d0; background: #ecfdf5;` 녹색 약하게
- 필터: `교과 / 학년 / 학기 / 매핑상태(전체|미매핑|부분|완료) / 키워드`
- "미매핑만 보기" 토글 추가 — 클릭 1번으로 720개 고립 노드만 노출
- 정렬: 기본 학습맵 순서, 옵션으로 매핑수↓ / 키워드 가나다

#### 중앙 — 매핑 상세 (1fr)
- 노드 헤더에 **진행률 바** 추가 (영상 0/2 · 문항 1/3 → 진행률 33%)
- 영상/문항 탭은 유지하되, **카드 좌측 4px 컬러 스트라이프**로 종류 즉시 식별:
  - 영상: `border-left: 4px solid #3b82f6;`
  - 문항: `border-left: 4px solid #10b981;`
- "콘텐츠 추가" 버튼을 우측 패널과 통합 → 사용자는 **우측에서 추천을 끌어다 가운데로** 드롭하거나 [추가] 버튼 클릭
- **빈 상태 CTA 강화**:
  ```
  ┌──────────────────────────────────────┐
  │      매핑된 영상이 아직 없어요.       │
  │                                      │
  │   ☆ AI 추천 5개 보기   ＋ 직접 추가  │
  └──────────────────────────────────────┘
  ```

#### 우측 — 콘텐츠 패널 (380px 고정)
- 3개 탭(`☆ 추천 / 🔍 검색 / # ID`)
- **추천이 기본 활성** — `lesson_name`, `unit_name`, `achievement_code`를 키워드로 EBS/YouTube/공개문항 자동 검색
- 각 추천 카드:
  ```
  ┌─────────────────────────────────────┐
  │ [썸네일]  영상 · EBS                 │
  │ 분수의 덧셈과 뺄셈                   │
  │ 정확도 92% (제목 키워드 4/5 일치)    │
  │              [미리보기]  [추가]      │
  └─────────────────────────────────────┘
  ```
- 검색·ID 직접 입력은 기존 모달 로직을 재활용하되 사이드 패널로 이동(모달 폐기 → CLAUDE.md z-index 위반 회피)

### 안 2. 자동 매핑 도구

#### 2-1. 단일 노드 추천 (1클릭)
- 노드 선택 시 우측 추천 패널 자동 호출
- API: `GET /api/admin/learning-map/nodes/:id/recommendations` (신규)
  - 입력: 노드 lesson_name + unit_name + achievement_code
  - 처리: 공개 콘텐츠 제목·태그 키워드 매칭 점수 계산 (TF-IDF or 간이 토큰 매칭)
  - 출력: 정확도 0~100%로 정렬된 추천 콘텐츠 N개 (영상 5 + 문항 10)

#### 2-2. 일괄 자동 매핑 (필터 범위)
- "현재 필터 범위 자동 매핑" 버튼 (좌측 상단)
- 동작: 필터에 걸린 N개 미매핑 차시에 대해 추천 실행
  - **정확도 ≥ 80%**: 자동 적용 (배지 "AI" 표시 — 검수 대기)
  - **정확도 60~80%**: 검수 큐로 이동 (관리자 1클릭 승인/거부)
  - **정확도 < 60%**: 적용 안 함, "추천 없음"으로 표시
- 진행 모달: `프로그레스 바 + 처리 N/M + 취소`

#### 2-3. 검수 큐
- 좌측 트리 위 작은 띠: `검수 대기 23건 [열기]`
- 클릭 시 가운데에 검수 카드 슬라이드:
  ```
  ┌─ 검수 카드 ──────────────────┐
  │ 노드: 3학년 1학기 1단원 2차시 │
  │ 추천된 영상: "분수의 시작"    │
  │ 정확도 71%                    │
  │ [✓ 적용]  [✕ 거부]  [건너뜀]  │
  └─────────────────────────────┘
  ```

### 안 3. CSV/Excel 가져오기·내보내기

#### 3-1. 내보내기
- 우상단 `[CSV ↓ 내보내기]` 버튼
- 형식: `node_id, subject, grade, semester, unit, lesson, achievement_code, role, content_id, content_title, sort_order`
- 현재 매핑 전체 백업 (운영팀이 Excel에서 검토)

#### 3-2. 가져오기
- 우상단 `[CSV ↑ 가져오기]` 버튼 → 모달
- 단계:
  1. 파일 업로드
  2. **드라이런 검증** — `node_id` 미존재 / `content_id` 미존재 / 중복 / 형식 오류 표 노출
  3. 승인 시 적용 (atomic transaction)
- 결과 리포트: `삽입 N · 갱신 N · 스킵 N (이유별)` — 학습맵 Excel 업로드와 동일 패턴

### 안 4. 매핑 통계 대시보드 카드

상단(필터바 위) 4-칩 그리드:

```
┌──────────┐┌──────────┐┌──────────┐┌──────────┐
│ 매핑률   ││ 영상     ││ 문항     ││ 고립노드 │
│ 38%      ││ 312건    ││ 580건    ││ 720개    │
│ 442/1162 ││ 평균 0.7 ││ 평균 1.3 ││ [열기]   │
└──────────┘└──────────┘└──────────┘└──────────┘
```

- "고립 노드 [열기]" 클릭 → 좌측 필터에 "미매핑만 보기" 자동 적용
- "과잉 매핑 (10+ 매핑)" 카드 추가 가능 — 클릭 시 콘텐츠 정리 모드

추가 — 영역별 매핑률 표 (`lmSubjectStats` 카드 확장):

| 교과 | 노드 | 매핑완료 | 비율 |
|------|------|----------|------|
| 국어 | 256 | 210 | 82% |
| 수학 | 312 | 110 | 35% |
| ... | ... | ... | ... |

비율 셀에 **데이터 바**(0~100% 너비) 표시 — `linear-gradient(90deg, #c4b5fd, transparent)`.

| 모달 z-index ≥ 10000 | 모달 폐기 → 사이드패널로 이동, 미리보기 iframe만 모달 유지 (z-index `10010`) |

| 복귀(c) | 단원 drill-down "← 돌아가기" 버튼, ESC 키 처리 |

## 7. 단계별 도입 로드맵 (개발 영향 최소)

| Phase | 기간 | 산출물 | 백엔드 의존도 |
|-------|------|--------|----------------|
| P1 | 1주 | 통계 칩 4개, 노드 카드 상태 색상, 빈 상태 CTA, 단원 돌아가기 버튼 | 낮음 (기존 API로 가능) |
| P2 | 2주 | 3-pane 레이아웃 + 우측 사이드패널(모달 폐기), 미매핑 필터 | 중간 (`mapping_status` 필터 1개 추가) |
| P3 | 2주 | AI 추천 (단일 노드) + 검수 큐 | 높음 (`/recommendations` API) |
| P4 | 2주 | 일괄 자동 매핑 + CSV 가져오기/내보내기 | 높음 (`/auto-map`, `/import`, `/export`) |
| P5 | 1주 | 학생 화면 미리보기 iframe + 변경 이력 | 낮음 |

P1만 적용해도 점검 보고서 Critical 4건 중 C4(빈 상태)·H1(미매핑 시각화) 해소.

## 8. 위험 / 미해결 질의

1. **AI 추천 정확도 보정** — `lesson_name` 키워드 매칭만으로 80%를 초과할지 콘텐츠 메타 품질에 따라 다름. 콘텐츠에 `tags`, `achievement_codes` 컬럼 추가 필요할 수 있음.
2. **CSV 가져오기 권한** — 잘못된 가져오기로 매핑 전체 손상 가능 → 기본 `dry_run` + 1회 백업 강제.
3. **사이드패널 vs 모달** — 모바일/태블릿(<1024px)에서는 사이드패널이 좁아짐. 768px 미만은 모달로 폴백.
4. **변경 이력 보관 기간** — 90일 / 무제한 결정 필요.
5. **학생 미리보기 iframe** — 같은 도메인이라 OK. 인증 쿠키 공유 시 학생 시점 시뮬레이션 정확도 검증 필요.
