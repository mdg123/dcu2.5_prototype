## Critical / High / Medium / Low 이슈 분류

### Critical

#### C-1. `nodeLevel` 파라미터 API 미처리 — 단원/차시 혼합 목록 반환
- **위치**: `GET /api/admin/learning-map/nodes` (routes/admin.js line 254~297)
- **현상**: 프론트엔드의 "차시/단원" 전환 드롭다운(`lmFilterLevel`)이 `nodeLevel` 파라미터를 서버로 전송하지만, API에서 해당 파라미터를 완전히 무시한다. `req.query`에서 `nodeLevel`을 추출하는 코드가 없음.
- **결과**: 기본 "차시" 모드로 검색해도 단원 노드(level=2)가 함께 목록에 나타난다. 관리자가 단원을 클릭하면 drill-down 화면이 표시되지만, 그 의도가 불명확해 첫 방문자에게 혼란을 준다.
- **재현**: `?nodeLevel=3&grade=1` 파라미터로 API 호출 시, 응답에 node_level=2 노드가 그대로 포함됨.

#### C-2. 콘텐츠 검색에서 학년·교과 필터 완전 부재
- **위치**: `GET /api/admin/contents/public-search` (admin.js line 391~429)
- **현상**: `type`(video/question)과 `keyword`만 지원. `grade`, `subject` 파라미터를 전달해도 WHERE 절에 포함되지 않아 전체 공개 콘텐츠 목록에서 키워드만으로 검색해야 한다.
- **결과**: 선택한 노드(예: 1학년 수학 > 9까지의 수 알기)와 관련 없는 타 학년·타 교과 콘텐츠가 섞여 노출됨. 올바른 콘텐츠를 찾으려면 노드 컨텍스트를 기억하며 여러 번 검색을 반복해야 한다.

### High

#### H-1. 미매핑 노드 시각적 강조 없음
- 노드 카드에 `영상 0 / 문항 2` 배지가 표시되지만, 영상 미매핑(157개) 상태가 색상이나 아이콘으로 특별 강조되지 않는다. "위험/주의" 시각 신호가 없어 방치하기 쉽다.
- 통계 카드(stat-grid)에 "영상 미매핑 차시 수" 항목이 없다.

#### H-2. 콘텐츠 추가 모달에서 현재 노드 컨텍스트(학년·교과) 미표시
- 모달 상단에 "어떤 노드에 추가 중인지" 정보가 없다. 탭(영상/문항)과 검색창만 있어 관리자가 컨텍스트를 잃기 쉽다.
- 특히 단원 드릴다운 → 차시 클릭 시 상단 노드 헤더 영역이 업데이트되나, 모달 열린 뒤에는 확인 불가.

#### H-3. 매핑 삭제 시 확인 메시지 최소화 — 학습 활동 데이터 경고 없음
- `removeContentFromNode()`: `confirm('이 콘텐츠 매핑을 제거하시겠습니까?')` 단순 confirm만 존재.
- 해당 콘텐츠에 학생 학습 로그(user_node_status, learning_log 등)가 있어도 추가 경고가 없다. 실수로 매핑 삭제 시 학생 이력이 고아 상태가 될 수 있다.
- 참고: CASCADE는 node_contents 삭제이므로 contents 자체와 학습 로그는 유지되나, 학습경로 재계산 등 연쇄 영향에 대한 안내가 없다.

#### H-4. bulk 매핑 기능 전무
- 한 번에 여러 노드에 같은 콘텐츠를 매핑하거나, 여러 콘텐츠를 한 노드에 일괄 추가하는 기능이 없다.
- 157개 영상 미매핑 차시를 처리하려면 노드 선택 → 모달 열기 → 검색 → 추가 → 모달 닫기를 157회 반복해야 한다.

#### H-5. 교과명 중복 이슈 ('수학' vs '수학과')
- DB에 subject = '수학' 노드 1,256개, '수학과' 52개가 혼재한다.
- 필터 드롭다운에 두 항목이 분리 노출(수학 (1256), 수학과 (52)). 관리자가 '수학과'를 선택하면 누락된 46개 차시가 별도 영역처럼 보인다.
- 학습 맵 업로드 시 정규화 로직이 없는 것으로 추정.

### Medium

#### M-1. 자동 추천 기능 없음
- 노드의 `lesson_name`, `unit_name`, `achievement_code`를 기반으로 관련 콘텐츠를 자동 추천하는 기능이 없다. 콘텐츠 검색 API가 노드 컨텍스트를 인지하지 못한다.

#### M-2. 노드 목록 기본값 — 초기 진입 시 빈 상태
- 학습맵 관리 섹션 진입 시 노드 목록에 "필터를 선택하고 검색하세요"만 표시. 자동 로드가 없어 필터를 선택해야 데이터가 보인다. (단, `loadLearningMapSummary()`에서 통계는 자동 로드됨.)

#### M-3. 순서 변경(↑↓)이 두 번의 별도 API 호출
- `moveContentOrder()`: 두 콘텐츠의 sort_order를 교환하기 위해 `PUT` 요청 2회를 순차 실행. 하나가 실패해도 부분 변경될 수 있어 데이터 정합성 위험 존재. 단일 트랜잭션 API 엔드포인트 필요.

#### M-4. 공개 콘텐츠 검색 결과 최대 30개 (default) / 100개 (limit 상한)
- `public-search` API의 `limit` 최대값이 100으로 제한(line 395). 공개 콘텐츠가 2,611개인데 페이지네이션 없이 30개만 기본 노출. 검색어 없이 목록 전체를 볼 수 없다.

#### M-5. CSV/Excel 매핑 내보내기/가져오기 기능 없음
- 현재 매핑 현황을 Excel로 다운로드하거나, 미리 준비한 매핑 데이터를 일괄 업로드하는 기능이 없다.

#### M-6. 작업 이력/로그 없음
- 어떤 관리자가 언제 어떤 노드에 콘텐츠를 매핑·해제했는지 기록이 없다. `node_contents` 테이블에 `created_at` 및 `created_by` 컬럼이 있더라도 UI에서 조회 불가.

### Low

#### L-1. 단원 드릴다운 화면에서 다시 노드 목록으로 돌아가는 명시적 "뒤로" 버튼 없음
- 단원 클릭 후 드릴다운 차시 그리드가 표시되나, 원래 노드 목록으로 돌아가려면 좌측 패널에서 다른 노드를 클릭해야 한다.

#### L-2. 매핑 순서가 영상/문항 탭별로 독립 관리되나 sort_order 기준 모호
- 검색 모달에서 추가한 콘텐츠의 sort_order가 `current.length + 1`로 자동 할당(line 1464). 기존 sort_order와 충돌 가능.

#### L-3. ID 직접 입력 탭에서 콘텐츠 미리보기 없음
- 숫자 ID를 입력해도 콘텐츠 제목·타입 확인 없이 바로 추가된다. 잘못된 ID 입력 시 서버 응답(404) 전까지 오류를 알 수 없다.

#### L-4. 모달 열린 상태에서 노드 목록 클릭 불가 (overlay)
- 모달 오버레이가 떠 있는 동안 좌측 노드 패널을 클릭할 수 없어, 다른 노드로 전환하려면 반드시 모달을 닫아야 한다.

## "맵핑을 편리하게" — 사용자 관점 평가

### 처음 보는 관리자가 5분 안에 1개 콘텐츠를 노드에 매핑할 수 있는가?

**Yes — 단, 조건부**

순서: 좌측 메뉴 "학습맵 관리" 클릭 → 교과·학년 필터 선택 → 검색 → 차시 노드 클릭 → "콘텐츠 추가" 버튼 → 키워드 검색 → "추가" 버튼. 총 7~8번의 조작으로 가능하며, 각 단계가 레이블로 안내되어 있어 직관적이다.

조건부 이유: "차시" 모드로 필터링해도 단원 노드가 섞여 나오므로(C-1 버그), 처음 보는 관리자가 단원 카드를 클릭해 드릴다운 화면으로 빠진 후 혼란을 겪을 수 있다.

### 100개 콘텐츠를 1,162개 노드에 매핑하려면 몇 클릭이 필요한가? (정량 추정)

**매핑 1건당 최소 클릭 수 (이미 차시가 화면에 보인다고 가정)**:
1. 노드 카드 클릭 (1)
2. "콘텐츠 추가" 버튼 (1)
3. 검색어 입력 (키보드 입력, 클릭 포함) + 검색 버튼 (2)
4. 검색 결과에서 "추가" 버튼 (1)
5. 모달 "닫기" 버튼 (1)

= 최소 6회 클릭/조작

**157개 영상 미매핑 차시를 모두 처리하는 경우 (각 노드마다 해당 콘텐츠 키워드 검색 필요)**:
- 노드 필터링·스크롤·클릭 포함 시 노드당 약 8~10 조작
- 157개 × 9회 평균 = **약 1,413번 조작**
- 소요 시간 추정: 노드당 1~2분 × 157 = **약 2.6~5.2시간**

**1,162개 노드 전체 매핑 갱신 시 (노드당 1개 콘텐츠 교체)**:
- 약 1,162 × 9 = **약 10,458번 조작** (현실적으로 불가능한 수준)

### 가장 짜증나는 동선 Top 3

**1위: 차시를 찾기 위한 반복적인 단계 (모달 열기 → 검색 → 닫기 × N)**
단원이 노드 목록에 섞여 나와 실수로 클릭하면 드릴다운으로 빠지고, 다시 좌측 목록을 스크롤해야 한다. 차시 필터가 API에서 동작하지 않아 "차시" 선택 의미가 없다.

**2위: 콘텐츠 검색에서 학년·교과 무관 전체 콘텐츠 노출**
예를 들어 "1학년 수학 > 9까지의 수 알기" 노드를 선택했는데, 모달에서 키워드 없이 검색하면 전혀 다른 학년의 타 교과 영상들이 뜬다. 노드 컨텍스트가 검색에 전달되지 않아 관리자가 수동으로 맥락을 기억하며 키워드를 입력해야 한다.

**3위: 미매핑 157개 영상 차시를 한눈에 파악할 방법 없음**
통계 카드에 "영상 미매핑 차시 157개" 같은 수치가 없고, 노드 목록에서 일일이 스크롤하며 "영상 0" 배지를 찾아야 한다. 미매핑 노드만 모아 보거나 필터링하는 기능이 없다.

## 우선 개선 요청 Top 5

### P1. `nodeLevel` API 필터 구현 (Critical, 즉시 수정)
`routes/admin.js`의 `GET /api/admin/learning-map/nodes`에 `nodeLevel` 파라미터 처리 추가.
```js
const nodeLevel = req.query.nodeLevel ? parseInt(req.query.nodeLevel) : null;
if (nodeLevel) { where += ' AND n.node_level = ?'; params.push(nodeLevel); }
```
예상 효과: 차시 모드 선택 시 단원 카드 혼재 해소, 첫 방문자 혼란 제거.

### P2. 콘텐츠 검색 API에 노드 컨텍스트(학년·교과·성취기준코드) 필터 추가 (Critical)
`public-search` API에 `grade`, `subject`, `achievement_code` 파라미터 추가. 프론트엔드에서 현재 선택된 노드 정보를 모달 열기 시 자동 전달. 모달 상단에 "대상 노드" 컨텍스트 표시.

### P3. 미매핑 노드 강조 필터 (High)
노드 목록에 `mappingFilter=empty_video` 옵션 추가(API: `WHERE videos_count = 0`). 통계 카드에 "영상 미매핑 차시 157개" 클릭 시 해당 필터로 자동 이동. 영상 미매핑 카드에 주황색 좌측 보더 표시.

### P4. Excel/CSV 매핑 일괄 작업 (High)
현재 매핑 현황을 CSV로 내보내기(node_id, lesson_name, content_id, content_type 컬럼). 반대로 (node_id, content_id, role) CSV를 업로드하면 일괄 매핑 처리. 이 기능 하나로 157개 영상 미매핑 문제를 수분 내 해결 가능.

### P5. 교과명 정규화 (High) + 매핑 삭제 경고 강화 (High)
Excel 업로드 시 '수학과' → '수학' 자동 정규화(또는 정규화 테이블 참조). 매핑 삭제 전 해당 콘텐츠의 학습 이력 수(view_count, log_count)를 조회해 "학생 N명이 학습한 콘텐츠입니다. 계속하시겠습니까?" 경고 추가.
