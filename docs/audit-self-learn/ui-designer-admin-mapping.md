## 0. 한 줄 결론

> "맵핑을 편리하게"라는 사용자 요구에 비해 **현재 UI는 1노드-1콘텐츠 매핑에만 최적화된 단방향 드릴다운**이고, 일괄/추천/검색/통계가 모두 부재하여 **1,162개 노드를 채우려면 평균 8~10클릭 × N회**의 선형 작업을 요구한다. 워크스페이스 재설계와 자동 추천이 없으면 운영 단계에서 매핑률이 정체된다.

## 2. "편리한 맵핑" 평가

### 현재 동선 클릭 수 측정 (코드 추적 결과)

#### 시나리오 A — 노드 1개에 영상 1개 매핑

| 단계 | 사용자 행동 | 클릭 |
|------|-------------|------|
| 1 | 좌측 메뉴 "학습맵 관리" 클릭 | 1 |
| 2 | 노드-콘텐츠 매핑 카드까지 스크롤 | 0 (스크롤) |
| 3 | 교과 선택 | 1 |
| 4 | 학년 선택 | 1 |
| 5 | (선택) 학기 선택 | 1 |
| 6 | 키워드 입력 | 타이핑 |
| 7 | "검색" 클릭 | 1 |
| 8 | 좌측 노드 카드 1개 클릭 | 1 |
| 9 | (영상 탭 기본 활성) "콘텐츠 추가" 클릭 | 1 |
| 10 | 모달 키워드 입력 | 타이핑 |
| 11 | 모달 "검색" 클릭 | 1 |
| 12 | 결과 행의 "추가" 클릭 | 1 |
| 13 | 모달 "닫기" 클릭 | 1 |

**합계: 10클릭 + 2회 타이핑** (현재 모달은 추가 후 자동으로 안 닫히고, 추가 토스트만 뜸)

#### 시나리오 B — 노드 1개에 영상 3개 매핑

10클릭 + (타이핑) + (검색→추가→재검색→추가→재검색→추가) ≈ **18클릭 + 4회 타이핑**

#### 시나리오 C — 100개 노드에 각각 1개씩 매핑

100 × (시나리오 A의 노드 선택 이후 6클릭) ≈ **600클릭 이상**. 도구 부재로 일괄 처리 불가.

#### 시나리오 D — 미매핑 노드 찾기

**불가능**. 현재 필터에 매핑 여부 없음. 노드 카드 표시되는 `영상 N · 문항 M` 배지를 사람 눈으로 0인 것만 골라야 함.

### 가독성 · 시각 일관성 평가

- **일관성 양호**: 보라(#7c3aed) 액센트, `badge-student` 파랑/`badge-teacher` 녹색 배지가 학생 화면과 동일.
- **일관성 위반**:
  - 알림 톤 — 추가/제거는 토스트, 추가 실패는 alert, 제거는 confirm. 통일 필요.
  - 필터바 입력 padding `6px 8px` (line 99) vs 본문 입력 `8px 10px` (line 446) — 폼 스케일 불일치.
  - 모달 닫기 버튼은 우상단(`modal-close`)인데, 매핑 카드의 제거 버튼(`lm-ibtn.danger`)은 우측 액션 그룹에 있음. 기능은 같은데 위치·크기·색상 코드가 모두 다름.
  - 단원 drill-down 시 헤더 영역이 `border:1px dashed #f59e0b; background:#fffbeb` (주황색)로 변하나, 그 외 어떤 상태에서도 주황은 등장하지 않음 — 신규 색상 1개 추가 비용.

| 매핑 추가 후 학생 드로어 반영 확인 | **시각 신호·미리보기 모두 없음** | 사용자가 매핑 결과를 학생 화면에서 직접 열어 확인해야 함 |
